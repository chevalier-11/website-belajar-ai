9.1 Mengimpor Modul Bawaan Python

Python hadir dengan baterai sudah terpasang *(batteries included)* — begitu filosofi resminya. Artinya, Python sudah menyertakan ratusan modul siap pakai yang bisa kamu impor tanpa menginstal apapun. Modul-modul ini disebut pustaka standar *(standard library)*, dan mereka menangani segalanya mulai dari matematika, pengelolaan tanggal dan waktu, akses sistem berkas, komunikasi jaringan, hingga ekspresi reguler.

Untuk menggunakan isi sebuah modul, kamu perlu mengimpornya terlebih dahulu dengan pernyataan import. Ada beberapa cara mengimpor dan masing-masing punya kelebihan yang cocok untuk situasi yang berbeda. Memilih cara yang tepat bukan soal teknis — melainkan soal membuat kode mudah dibaca dan menghindari konflik nama:

python — empat cara mengimpor 
# Cara 1: import seluruh modul — akses via nama.fungsi
import math
print(math.sqrt(144))    # → 12.0
print(math.pi)            # → 3.141592653589793
print(math.floor(3.7))   # → 3

# Cara 2: import dengan alias — berguna untuk nama panjang
import datetime as dt
sekarang = dt.datetime.now()
print(sekarang.strftime("%Y-%m-%d %H:%M"))

# Cara 3: impor hanya yang dibutuhkan — langsung bisa dipakai tanpa awalan
from math import sqrt, pi, ceil
print(sqrt(25))    # → 5.0 — tidak perlu 'math.' di depan
print(pi)          # → 3.141592...

# Cara 4: from ... import * — JANGAN lakukan ini
# from math import *  ← mengimpor semua nama, bisa tumpang tindih!
⚠️
Hindari "from modul import *"

Mengimpor semua nama dari sebuah modul dengan tanda bintang terlihat praktis tapi berbahaya. Kamu tidak tahu nama apa saja yang masuk, dan ada risiko nama dari modul tersebut menimpa fungsi atau variabel yang sudah kamu definisikan. Kode yang memakai ini juga sulit dibaca — pembaca tidak tahu fungsi tertentu berasal dari mana. Gunakan cara 1, 2, atau 3 saja.

Pelajari setidaknya yang ada di tabel ini — kamu tidak perlu hafal semua fungsinya, tapi penting tahu modul mana yang mengurus masalah apa agar kamu bisa langsung tahu ke mana harus mencari:

data = list(range(100))

print(f"Train: {len(train)}, Test: {len(test)}")

# ── os.path: penanganan path yang aman lintas sistem operasi ───
folder_data  = "dataset"
nama_file    = "nilai_siswa.csv"
path_lengkap = os.path.join(folder_data, nama_file)

# Cek keberadaan sebelum membuka

    print("File ditemukan!")
else:
    print("File tidak ada — buat dulu!")

# Buat folder jika belum ada
os.makedirs("output/laporan", exist_ok=True)   # exist_ok: tidak error jika sudah ada

# ── json: simpan dan muat konfigurasi ─────────────────────────
config = {
    "model": "XGBoost",
    "params": {"lr": 0.05, "depth": 6, "n_estimators": 200},
    "tanggal": "2026-02-26"

# ── time: ukur berapa lama kode berjalan ──────────────────────

pathlib — Cara Modern Menangani Path

Modul pathlib (masuk sejak Python 3.4) menawarkan cara yang jauh lebih elegan untuk bekerja dengan path file dibandingkan os.path yang lama. Dengan pathlib, path adalah objek dengan metode, bukan sekadar string. Kode menjadi lebih bersih, lebih mudah dibaca, dan lebih mudah diuji:

python — pathlib modern 
from pathlib import Path

# Buat objek Path
folder_proyek = Path("proyek_ml")
file_data     = folder_proyek / "data" / "raw" / "siswa.csv"
# operator / membuat path baru — jauh lebih bersih dari os.path.join()

print(file_data)                     # → proyek_ml/data/raw/siswa.csv
print(file_data.name)                # → siswa.csv
print(file_data.stem)                # → siswa (tanpa ekstensi)
print(file_data.suffix)              # → .csv
print(file_data.parent)              # → proyek_ml/data/raw

# Buat direktori (dan semua induknya jika belum ada)
folder_output = Path("proyek_ml/output/grafik")
folder_output.mkdir(parents=True, exist_ok=True)

# Temukan semua file CSV dalam folder secara rekursif
folder_data = Path(".")   # folder saat ini
semua_csv = list(folder_data.glob("**/*.csv"))
print(f"Ditemukan {len(semua_csv)} file CSV")

# Tulis dan baca file — langsung dari objek Path
file_catatan = Path("catatan.txt")
file_catatan.write_text("Eksperimen pertama berhasil!\n", encoding="utf-8")
isi = file_catatan.read_text(encoding="utf-8")
print(isi.strip())

9.2 Menginstal dan Menggunakan Paket Eksternal dengan pip

Pustaka standar Python sudah hebat, tapi ia punya batasan yang disengaja — hanya menyediakan alat-alat generik. Untuk pekerjaan spesifik seperti analisis data, pembelajaran mesin, atau visualisasi, kamu butuh paket-paket dari komunitas yang dikembangkan oleh orang-orang di seluruh dunia. Paket-paket ini tersedia gratis di PyPI *(Python Package Index)* — repositori terpusat di pypi.org yang saat ini menyimpan lebih dari 500.000 paket.

Alat untuk menginstal paket dari PyPI adalah pip — package installer for Python. Ia sudah terinstal otomatis bersama Python modern. Kamu menggunakannya dari terminal, bukan dari dalam skrip Python:

Lingkungan Virtual (Virtual Environment) — Wajib Dipahami

Menginstal semua paket ke instalasi Python utama *(global)* adalah cara yang baik untuk merusak proyek-proyekmu di masa depan. Masalahnya adalah konflik versi: proyek A butuh NumPy versi 1.24, tapi proyek B butuh versi 2.0. Kalau keduanya berbagi satu instalasi Python, salah satunya pasti tidak bisa berjalan dengan benar.

Solusinya adalah lingkungan virtual *(virtual environment)* — sebuah folder terisolasi yang punya salinan Python dan koleksi paket sendiri, terpisah dari Python global dan dari proyek lain. Setiap proyek punya lingkungan virtualnya sendiri. Python sudah menyertakan modul venv untuk membuat ini, tanpa perlu menginstal apapun:

$ terminal — membuat dan menggunakan venv
# Langkah 1: Buat lingkungan virtual di folder proyek
$ cd proyek_ml
$ python -m venv .venv        # buat folder .venv di dalam proyek

# Langkah 2: Aktifkan lingkungan virtual
$ source .venv/bin/activate   # Linux / macOS
$ .venv\Scripts\activate       # Windows (Command Prompt)
$ .venv\Scripts\Activate.ps1  # Windows (PowerShell)

# Tanda lingkungan aktif: nama venv muncul di prompt
(.venv) $

# Langkah 3: Instal paket — hanya masuk ke lingkungan ini
(.venv) $ pip install numpy pandas scikit-learn

# Langkah 4: Simpan daftar paket yang terinstal
(.venv) $ pip freeze > requirements.txt

# Langkah 5: Nonaktifkan saat selesai
(.venv) $ deactivate
$
💡
requirements.txt — Kunci Reprodusibilitas

File requirements.txt mencatat daftar semua paket beserta versinya yang dipakai proyek. Siapapun yang mengkloning proyekmu bisa langsung menginstal semua dependensi yang sama dengan satu perintah: pip install -r requirements.txt. Ini adalah praktik standar industri — tanpa file ini, orang lain (atau dirimu sendiri di komputer lain) harus menebak paket apa saja yang dibutuhkan.

9.3 Membuat Modul Sendiri

Modul Python tidak lebih dari sebuah file .py yang berisi fungsi, kelas, dan variabel yang ingin kamu bagikan antar file. Saat proyekmu mulai punya lebih dari satu file, ini menjadi cara utama untuk menghindari duplikasi kode — kamu menulis logika sekali di satu modul, lalu mengimpornya dari mana saja yang membutuhkannya.

Tidak ada sintaksis khusus untuk "membuat modul" — cukup buat file .py dengan nama yang deskriptif, isi dengan fungsi atau kelas yang relevan, dan kamu sudah punya modul. Kuncinya adalah konvensi: isi modul harus kohesif — satu modul menangani satu kategori tanggung jawab, bukan campuran dari berbagai hal yang tidak berkaitan.

python — statistik_utils.py (modul yang kita buat) 
# File: statistik_utils.py
# Modul ini berisi fungsi-fungsi statistik untuk keperluan ML

"""
Modul utilitas statistik dasar.

Fungsi-fungsi ini menerima list angka dan mengembalikan
berbagai metrik statistik tanpa bergantung pada library eksternal.
"""

def rata_rata(data):
    """Hitung mean dari list angka."""
    if not data:
        raise ValueError("Data tidak boleh kosong")
    return sum(data) / len(data)

def varian(data, populasi=False):
    """
    Hitung varian dari list angka.

    Parameter:
        data      (list) : list angka
        populasi  (bool) : True = varian populasi, False = varian sampel (default)
    """
    if len(data) < 2:
        raise ValueError("Butuh minimal 2 data untuk menghitung varian")
    mu     = rata_rata(data)
    pembagi = len(data) if populasi else len(data) - 1
    return sum((x - mu) ** 2 for x in data) / pembagi

def std_dev(data, populasi=False):
    """Hitung simpangan baku (standar deviasi)."""
    return varian(data, populasi) ** 0.5

def normalisasi_min_max(data):
    """Normalisasi data ke rentang [0, 1]."""
    minimum = min(data)
    maksimum = max(data)
    rentang  = maksimum - minimum
    if rentang == 0:
        return [0.0] * len(data)
    return [(x - minimum) / rentang for x in data]

def ringkasan(data):
    """Tampilkan ringkasan statistik dasar."""
    print(f"  n       : {len(data)}")
    print(f"  min     : {min(data)}")
    print(f"  max     : {max(data)}")
    print(f"  mean    : {rata_rata(data):.4f}")
    print(f"  std dev : {std_dev(data):.4f}")

# Variabel level modul — bisa diakses dari luar
VERSI = "1.0.0"

Setelah file statistik_utils.py ada, kamu bisa mengimpornya dari file lain yang berada di folder yang sama. Perhatikan bahwa nama modul adalah nama file tanpa ekstensi .py:

python — analisis.py (menggunakan modul kita) 
# File: analisis.py — mengimpor modul buatan sendiri
import statistik_utils as su    # impor modul dengan alias

nilai_ujian = [78, 85, 92, 67, 88, 91, 74, 83]

print("=== Ringkasan Nilai Ujian ===")
su.ringkasan(nilai_ujian)

print(f"\nVersi modul: {su.VERSI}")

# Impor hanya fungsi yang dibutuhkan
from statistik_utils import normalisasi_min_max, std_dev

nilai_norm = normalisasi_min_max(nilai_ujian)
print(f"\nNilai ternormalisasi:")
for asli, norm in zip(nilai_ujian, nilai_norm):
    print(f"  {asli} → {norm:.3f}")

print(f"\nStd dev: {std_dev(nilai_ujian):.2f}")

Blok if __name__ == "__main__"

Saat Python mengimpor sebuah modul, ia menjalankan seluruh kode yang ada di level atas modul tersebut. Ini bisa jadi masalah jika modul punya kode pengujian atau demonstrasi yang tidak ingin kamu jalankan saat diimpor — kamu hanya ingin kode itu berjalan ketika file tersebut dijalankan langsung. Pola if __name__ == "__main__" adalah solusinya:

python — statistik_utils.py dengan blok main 
# ... (kode fungsi di atas) ...

# Blok ini HANYA dijalankan saat file dieksekusi langsung:
# python statistik_utils.py
# TIDAK dijalankan saat diimpor: import statistik_utils
if __name__ == "__main__":
    print("=== Uji Coba Modul statistik_utils ===")
    data_uji = [10, 20, 30, 40, 50]

    print(f"Rata-rata: {rata_rata(data_uji)}")        # → 30.0
    print(f"Std dev  : {std_dev(data_uji):.4f}")     # → 15.8114
    print(f"Normalisasi: {normalisasi_min_max(data_uji)}")
    # → [0.0, 0.25, 0.5, 0.75, 1.0]

    print("\n--- Uji Ringkasan ---")
    ringkasan(data_uji)
    print("Semua fungsi berjalan normal.")
✅
Selalu Tambahkan Blok if __name__ == "__main__" di Modulmu

Ini adalah kebiasaan yang sangat baik. Ia memungkinkan setiap modul berfungsi ganda: bisa diimpor sebagai pustaka oleh file lain, sekaligus bisa dijalankan langsung untuk keperluan pengujian atau demonstrasi cepat. Tanpa pola ini, kode uji di level atas modul akan ikut berjalan setiap kali modul tersebut diimpor, yang bisa memperlambat startup dan menyebabkan efek samping yang tidak diinginkan.

9.4 Struktur Proyek Python yang Rapi

Satu file Python sudah cukup untuk latihan kecil. Tapi begitu proyekmu punya beberapa komponen — data, kode preprocessing, kode model, skrip pelatihan, dan hasil eksperimen — menyimpan semuanya dalam satu folder datar tanpa organisasi akan cepat jadi kekacauan. Struktur folder yang rapi bukan sekadar estetika: ia membuatmu lebih cepat menavigasi kode, lebih mudah berkolaborasi, dan lebih mudah menjelaskan proyekmu ke orang lain atau calon atasan.

Tidak ada satu standar universal yang berlaku untuk semua jenis proyek Python, tapi ada pola yang sudah terbukti bekerja dengan baik untuk proyek data science dan ML. Pola ini dipakai secara luas di industri dan sangat kompatibel dengan GitHub:

File __init__.py — Menjadikan Folder sebagai Paket

Ketika kamu menaruh file bernama __init__.py di dalam sebuah folder, Python memperlakukan folder itu sebagai paket — artinya isinya bisa diimpor seperti modul. File __init__.py bisa kosong sama sekali (itu sudah cukup untuk menjadikan folder sebagai paket), atau bisa berisi kode inisialisasi dan ekspor untuk memudahkan impor dari luar:

python — src/__init__.py (opsional tapi berguna) 
# File: src/__init__.py
# Mengekspor fungsi-fungsi yang paling sering dipakai
# sehingga pengguna bisa menulis: from src import muat_data
# alih-alih:                      from src.preprocessing import muat_data

from .preprocessing import muat_data, bersihkan_data
from .evaluasi      import hitung_akurasi, laporan_klasifikasi

__version__ = "0.1.0"
__author__  = "Nama Kamu"
python — main.py (titik masuk proyek) 
# File: main.py — titik masuk utama proyek
# Jalankan: python main.py

from pathlib import Path
from src import muat_data, bersihkan_data, hitung_akurasi
from src.model import ModelKlasifikasi
from src.visualisasi import plot_distribusi

def main():
    """Jalankan pipeline prediksi dari awal sampai akhir."""
    print("=== Proyek Prediksi Nilai Siswa ===")

    # 3. Latih model

if __name__ == "__main__":
    main()

Godaan yang sering dialami pemula adalah menunda penataan proyek — "nanti saja kalau sudah besar". # File: src/preprocessing.py
import csv
from pathlib import Path

def muat_csv(path):
    """Muat file CSV dan kembalikan list of dict."""
    path = Path(path)
    if not path.exists():
        raise FileNotFoundError(f"File tidak ditemukan: {path}")
    with path.open(encoding="utf-8") as f:
        return list(csv.DictReader(f))

def konversi_tipe(data, kolom_angka):
    """Konversi kolom yang ditentukan ke tipe float."""
    hasil    = []
    dilewati = 0
    for baris in data:
        try:
            baris_baru = dict(baris)
            for kol in kolom_angka:
                baris_baru[kol] = float(baris[kol])
            hasil.append(baris_baru)
        except (ValueError, KeyError):
            dilewati += 1
    if dilewati:
        print(f"[preprocessing] {dilewati} baris dilewati karena nilai tidak valid")
    return hasil

if __name__ == "__main__":
    # Uji coba cepat
    contoh = [
        {"nama": "Andi", "nilai": "85"},
        {"nama": "Budi", "nilai": "tidak_valid"},
        {"nama": "Citra", "nilai": "92"},
    ]
    hasil = konversi_tipe(contoh, ["nilai"])
    print(f"Berhasil: {len(hasil)} baris")
    print(hasil)

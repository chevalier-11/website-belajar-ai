4.1 List: Membuat, Mengakses, dan Memodifikasi

List adalah struktur data yang paling sering kamu gunakan di Python. Bayangkan list seperti sebuah rak dengan banyak laci bernomor — kamu bisa menaruh barang apapun di setiap laci, mengambilnya lewat nomornya, menambah laci baru di bagian belakang, atau mencabut laci mana saja. Dalam Python, list dibuat dengan kurung siku [] dan elemen-elemennya dipisahkan oleh koma.

Hal yang membuat list Python sangat fleksibel adalah ia bisa menampung nilai dengan tipe yang berbeda-beda dalam satu list yang sama — campuran angka, teks, boolean, bahkan list lain di dalamnya. Tapi dalam praktik kecerdasan buatan, kamu biasanya akan bekerja dengan list yang isinya bertipe seragam, misalnya list angka semua atau list teks semua.

python 
# Membuat list
nilai_ujian  = [85, 92, 78, 95, 88]
nama_kota    = ["Jakarta", "Bandung", "Surabaya"]
list_kosong  = []
list_campuran = ["Rina", 22, True, 3.14]  # bisa, tapi jarang ideal

# Panjang list
print(len(nilai_ujian))   # → 5
print(len(nama_kota))    # → 3
print(len(list_kosong))  # → 0

Mengakses Elemen List

Sama seperti string, list menggunakan indeks berbasis nol untuk mengakses elemennya. Indeks pertama adalah 0, dan indeks negatif bisa dipakai untuk mengakses dari belakang. Pengirisan *(slicing)* juga bekerja sama persis dengan string:

python 
buah = ["apel", "mangga", "jeruk", "pisang", "melon"]
#        [0]       [1]       [2]      [3]       [4]
#        [-5]      [-4]      [-3]     [-2]      [-1]

# Akses elemen tunggal
print(buah[0])    # → apel
print(buah[2])    # → jeruk
print(buah[-1])   # → melon (terakhir)
print(buah[-2])   # → pisang

# Pengirisan (slicing)
print(buah[1:4])   # → ['mangga', 'jeruk', 'pisang']
print(buah[:3])    # → ['apel', 'mangga', 'jeruk']
print(buah[2:])    # → ['jeruk', 'pisang', 'melon']
print(buah[::-1])  # → ['melon', 'pisang', 'jeruk', 'mangga', 'apel']

Memodifikasi List

Berbeda dari string, list bersifat bisa diubah *(mutable)*. Kamu bisa mengubah nilai elemen tertentu, menambah elemen baru, menghapus elemen, dan masih banyak lagi. Ini yang menjadikan list sangat cocok untuk data yang perlu dimanipulasi:

python 
angka = [10, 20, 30, 40, 50]

# Mengubah nilai di indeks tertentu
angka[2] = 99
print(angka)  # → [10, 20, 99, 40, 50]

# Menambah elemen di akhir
angka.append(60)
print(angka)  # → [10, 20, 99, 40, 50, 60]

# Menambah banyak elemen sekaligus
angka.extend([70, 80])
print(angka)  # → [10, 20, 99, 40, 50, 60, 70, 80]

# Menyisipkan elemen di posisi tertentu
angka.insert(1, 15)  # sisipkan 15 di indeks 1
print(angka[:4])     # → [10, 15, 20, 99, ...]

# Menghapus elemen berdasarkan nilai
angka.remove(99)
print(angka[:5])  # → [10, 15, 20, 40, 50]

# Menghapus dan mengambil elemen terakhir (atau indeks tertentu)
terakhir = angka.pop()     # ambil dan hapus elemen terakhir
kedua    = angka.pop(1)   # ambil dan hapus elemen di indeks 1
print(f"Diambil: {terakhir}, {kedua}")

Operasi dan Metode List Lainnya

python 
nilai = [88, 72, 95, 61, 85, 90]

# Pengurutan — .sort() mengubah list asli
nilai.sort()
print(nilai)  # → [61, 72, 85, 88, 90, 95]

nilai.sort(reverse=True)
print(nilai)  # → [95, 90, 88, 85, 72, 61]

# sorted() membuat list baru tanpa mengubah yang asli
nilai_asli  = [88, 72, 95, 61]
nilai_urut  = sorted(nilai_asli)
print(nilai_asli)  # → [88, 72, 95, 61]  — tidak berubah
print(nilai_urut)  # → [61, 72, 88, 95]  — list baru

# Agregasi dasar — sangat sering dipakai di ML
print(sum(nilai_asli))    # → 316
print(min(nilai_asli))    # → 61
print(max(nilai_asli))    # → 95

# Cek keanggotaan dengan 'in'
print(72 in nilai_asli)   # → True
print(100 in nilai_asli)  # → False

# Posisi elemen
print(nilai_asli.index(95))  # → 2 (indeks elemen 95)
print(nilai_asli.count(88))  # → 1 (berapa kali 88 muncul)
💡
List di Dunia Kecerdasan Buatan

Dalam proyek ML nyata, kamu akan menyimpan label kelas dalam list, menyimpan hasil prediksi model dalam list, menyimpan nilai loss selama pelatihan dalam list untuk kemudian divisualisasikan, dan membangun batch data sebagai list of list. List adalah wadah data yang paling fundamental — kuasai ini dengan benar sebelum lanjut.

4.2 Tuple: Kapan Pakai Tuple, Bukan List

Tuple sangat mirip dengan list — keduanya adalah urutan elemen yang bisa diakses lewat indeks, keduanya bisa menampung tipe data apapun, dan keduanya mendukung pengirisan. Perbedaan utamanya ada satu: tuple tidak bisa diubah setelah dibuat. Inilah yang disebut sifat tidak bisa diubah *(immutable)*.

Tuple dibuat dengan kurung biasa () — atau bahkan tanpa kurung sama sekali, cukup pisahkan nilai dengan koma. Ini sering bikin pemula kebingungan, jadi perhatikan contoh berikut:

python 
# Membuat tuple
koordinat    = (-6.2088, 106.8456)   # latitude, longitude Jakarta
dimensi_citra = (224, 224, 3)          # tinggi, lebar, channel warna
satu_elemen  = (42,)                   # perlu koma! tanpa koma bukan tuple
tanpa_kurung  = 1, 2, 3              # ini juga tuple yang valid

print(type(koordinat))    # → <class 'tuple'>
print(type(tanpa_kurung)) # → <class 'tuple'>
print(type((42)))         # → <class 'int'> — ini bukan tuple!

# Akses elemen — sama seperti list
print(koordinat[0])  # → -6.2088
print(koordinat[1])  # → 106.8456

# Mencoba mengubah tuple — akan error!
# koordinat[0] = 0.0  # → TypeError: 'tuple' object does not support item assignment

Membongkar Tuple (Tuple Unpacking)

Salah satu fitur tuple yang paling elegan adalah kemampuannya untuk "dibongkar" — nilainya bisa langsung ditugaskan ke beberapa variabel sekaligus dalam satu baris. Ini disebut tuple unpacking dan sangat sering muncul dalam kode Python profesional:

python 
# Tuple unpacking dasar
koordinat = (-6.2088, 106.8456)
lat, lon  = koordinat
print(f"Latitude: {lat}, Longitude: {lon}")
# → Latitude: -6.2088, Longitude: 106.8456

# Sangat berguna saat fungsi mengembalikan banyak nilai
def statistik(data):
    return min(data), max(data), sum(data) / len(data)

nilai = [85, 90, 78, 92]
minimum, maksimum, rata = statistik(nilai)
print(f"Min: {minimum}, Max: {maksimum}, Rata: {rata:.1f}")
# → Min: 78, Max: 92, Rata: 86.2

# Menukar nilai dua variabel — idiom khas Python
a, b = 10, 20
a, b = b, a   # swap tanpa variabel sementara!
print(a, b)   # → 20 10

Kapan Pakai Tuple, Bukan List?

Ada tiga situasi utama di mana tuple lebih tepat dari list. Pertama, ketika data tidak boleh berubah secara semantik — koordinat GPS, dimensi gambar, rentang tanggal, atau konstanta konfigurasi. Memilih tuple menyatakan niat kamu dengan jelas: "data ini tidak seharusnya diubah." Kedua, ketika kamu ingin menggunakan koleksi sebagai kunci dictionary — list tidak bisa digunakan sebagai kunci karena bisa berubah, tapi tuple bisa. Ketiga, tuple sedikit lebih hemat memori dan lebih cepat dari list untuk data yang sifatnya statis.

✅
Aturan Praktis List vs Tuple

Gunakan list ketika koleksi data kamu mungkin berubah — elemen perlu ditambah, dihapus, atau diurutkan ulang. Gunakan tuple ketika koleksi merepresentasikan sesuatu yang sifatnya tetap — koordinat, dimensi, pasangan nilai yang bermakna secara bersama. Kalau ragu, mulai dengan list. Ubah ke tuple jika kamu sadar data itu tidak boleh dimodifikasi.

4.3 Dictionary: Pasangan Kunci-Nilai

Dictionary adalah struktur data yang menyimpan data dalam bentuk pasangan kunci-nilai *(key-value pairs)*. Bayangkan seperti kamus sungguhan: kamu mencari sebuah kata (kunci), dan kamus memberitahumu artinya (nilai). Atau bayangkan kartu identitas: ada kunci seperti "nama", "usia", "kota" — dan masing-masing punya nilai yang sesuai.

Dictionary sangat cocok digunakan ketika kamu ingin mengakses data berdasarkan nama yang bermakna, bukan berdasarkan posisi. Ini membuatnya jauh lebih mudah dibaca dan digunakan dibanding list untuk data yang punya label. Di dunia kecerdasan buatan, dictionary sering muncul sebagai format data JSON, sebagai representasi satu baris dataset, atau sebagai tempat menyimpan hiperparameter model.

python 
# Membuat dictionary dengan kurung kurawal {}
mahasiswa = {
    "nama"     : "Rizky Pratama",
    "nim"      : "2021001234",
    "jurusan"  : "Teknik Informatika",
    "ipk"      : 3.75,
    "aktif"    : True
}

# Akses nilai berdasarkan kunci
print(mahasiswa["nama"])     # → Rizky Pratama
print(mahasiswa["ipk"])      # → 3.75

# Cara aman: .get() tidak error kalau kunci tidak ada
print(mahasiswa.get("email"))           # → None (tidak error)
print(mahasiswa.get("email", "N/A"))  # → N/A (nilai default)
⚠️
Akses Kunci yang Tidak Ada = Error

Mengakses kunci yang tidak ada dengan kurung siku dict["kunci_tidak_ada"] akan langsung menghasilkan KeyError. Selalu gunakan dict.get("kunci") atau periksa dulu dengan "kunci" in dict ketika kamu tidak yakin apakah kunci tersebut ada. Ini adalah sumber bug yang sangat umum bagi pemula.

Menambah, Mengubah, dan Menghapus Entri

python 
profil = {"nama": "Sari", "usia": 23}

# Menambah kunci baru
profil["kota"]  = "Yogyakarta"
profil["hobi"]  = ["membaca", "coding"]  # nilai bisa berupa list!
print(profil)

# Mengubah nilai yang sudah ada
profil["usia"] = 24
print(profil["usia"])  # → 24

# Menghapus entri
del profil["hobi"]
nilai_terhapus = profil.pop("kota")  # hapus dan kembalikan nilainya
print(f"Kota dihapus: {nilai_terhapus}")  # → Kota dihapus: Yogyakarta

# Cek keberadaan kunci
print("nama" in profil)   # → True
print("email" in profil)  # → False

Iterasi pada Dictionary

Kamu bisa mengiterasi dictionary dengan tiga cara berbeda tergantung apa yang kamu butuhkan — hanya kuncinya, hanya nilainya, atau keduanya sekaligus. Cara ketiga adalah yang paling sering dipakai dalam praktik:

python 
hiperparameter = {
    "learning_rate" : 0.001,
    "batch_size"    : 32,
    "epochs"        : 50,
    "dropout"       : 0.3
}

# Iterasi kunci saja
for kunci in hiperparameter.keys():
    print(kunci)

# Iterasi nilai saja
for nilai in hiperparameter.values():
    print(nilai)

# Iterasi kunci dan nilai sekaligus — paling sering dipakai
for kunci, nilai in hiperparameter.items():
    print(f"{kunci}: {nilai}")
# → learning_rate: 0.001
# → batch_size: 32
# → epochs: 50
# → dropout: 0.3

# Menggabungkan dua dictionary (Python 3.9+)
default  = {"epochs": 10, "lr": 0.01}
kustom   = {"epochs": 50, "momentum": 0.9}
gabungan = default | kustom   # kustom menimpa nilai yang sama di default
print(gabungan)
# → {'epochs': 50, 'lr': 0.01, 'momentum': 0.9}
📌
Dictionary dan Format JSON

Dictionary Python dan format JSON sangat mirip — keduanya menggunakan pasangan kunci-nilai. Ini bukan kebetulan: ketika kamu bekerja dengan API web, database NoSQL, atau konfigurasi model, data hampir selalu datang dalam format JSON yang langsung bisa dikonversi ke dictionary Python. Modul json bawaan Python membuat konversi bolak-balik antara keduanya sangat mudah.

4.4 Set: Koleksi Unik Tanpa Urutan

Set adalah koleksi yang tidak memiliki urutan dan tidak menyimpan duplikat. Setiap nilai hanya bisa muncul satu kali dalam sebuah set, tidak peduli berapa kali kamu mencoba menambahkannya. Set dibuat dengan kurung kurawal {} seperti dictionary, tapi tanpa pasangan kunci-nilai — atau bisa juga dengan fungsi set().

Set sangat berguna untuk dua hal utama: menghilangkan duplikat dari sekumpulan data, dan melakukan operasi himpunan matematika seperti irisan *(intersection)*, gabungan *(union)*, dan selisih *(difference)*. Di bidang kecerdasan buatan, set sering dipakai untuk membandingkan dua kumpulan label atau fitur.

python 
# Membuat set
buah_a = {"apel", "mangga", "jeruk", "apel"}  # "apel" duplikat diabaikan
print(buah_a)  # → {'apel', 'mangga', 'jeruk'} — urutan tidak terjamin

# Cara paling umum: hapus duplikat dari list
nilai_mentah  = [5, 3, 8, 3, 5, 9, 1, 8]
nilai_unik    = set(nilai_mentah)
print(nilai_unik)          # → {1, 3, 5, 8, 9}
print(len(nilai_unik))     # → 5

# Kembalikan ke list jika perlu urutan
nilai_bersih = sorted(nilai_unik)
print(nilai_bersih)  # → [1, 3, 5, 8, 9]

# Menambah dan menghapus elemen
buah_a.add("pisang")
buah_a.discard("mangga")  # discard tidak error meski elemen tidak ada
print(buah_a)

Operasi Himpunan

Kekuatan sejati set ada di operasi himpunan matematis. Ini sangat berguna ketika kamu perlu membandingkan dua kelompok data, misalnya memeriksa fitur mana yang ada di data latih tapi tidak ada di data uji, atau menemukan label yang bermasalah:

python 
fitur_latih = {"umur", "gaji", "pendidikan", "kota"}
fitur_uji   = {"umur", "gaji", "status"}

# Gabungan — semua fitur dari keduanya
print(fitur_latih | fitur_uji)
# → {'umur', 'gaji', 'pendidikan', 'kota', 'status'}

# Irisan — fitur yang ada di keduanya
print(fitur_latih & fitur_uji)
# → {'umur', 'gaji'}

# Selisih — fitur di latih yang tidak ada di uji
print(fitur_latih - fitur_uji)
# → {'pendidikan', 'kota'}  ← fitur ini hilang di data uji!

# Selisih simetris — fitur yang hanya ada di salah satu
print(fitur_latih ^ fitur_uji)
# → {'pendidikan', 'kota', 'status'}

# Cek keanggotaan — jauh lebih cepat dari list untuk data besar
print("kota" in fitur_latih)   # → True
print("status" in fitur_latih) # → False
⚡
Set Jauh Lebih Cepat untuk Pengecekan Keanggotaan

Ketika kamu perlu memeriksa apakah sebuah nilai ada dalam koleksi besar (ribuan atau jutaan elemen), gunakan set bukan list. Pengecekan x in set bekerja hampir seketika terlepas dari ukuran set, sementara x in list harus menelusuri satu per satu dari awal — semakin besar listnya, semakin lama. Ini perbedaan yang sangat signifikan dalam aplikasi nyata.

4.5 Memilih Struktur Data yang Tepat

Setelah mengenal keempat struktur data utama Python, pertanyaan yang sering muncul adalah: kapan saya harus pakai yang mana? Ini adalah keputusan yang sangat penting karena pilihan struktur data yang tepat tidak hanya membuat kode lebih mudah dibaca, tapi juga bisa membuat program berjalan jauh lebih cepat.

Struktur Bisa Diubah? Berurutan? Duplikat? Kapan Pakai
List [] ✅ Ya ✅ Ya ✅ Ya Koleksi data yang perlu diubah atau diurutkan
Tuple () ❌ Tidak ✅ Ya ✅ Ya Data tetap: koordinat, dimensi, konfigurasi
Dictionary {k:v} ✅ Ya ✅ Ya* Kunci: ❌ / Nilai: ✅ Data berlabel, pencarian cepat berdasarkan nama
Set {} ✅ Ya ❌ Tidak ❌ Tidak Menghilangkan duplikat, operasi himpunan

*Dictionary mempertahankan urutan penyisipan sejak Python 3.7 ke atas.

Skenario Nyata dan Pilihan yang Tepat

Mari kita lihat beberapa skenario konkret yang akan sering kamu hadapi dan pilihan struktur data yang paling masuk akal untuk masing-masing:

python — skenario nyata 
# SKENARIO 1: Menyimpan riwayat nilai loss selama pelatihan model
# → Pakai LIST karena perlu append setiap epoch dan mungkin di-plot
riwayat_loss = []
for epoch in range(10):
    loss = 1.0 / (epoch + 1)  # simulasi loss turun
    riwayat_loss.append(loss)

# SKENARIO 2: Menyimpan dimensi input gambar untuk model CNN
# → Pakai TUPLE karena nilai ini tidak boleh berubah
ukuran_input = (224, 224, 3)
tinggi, lebar, channel = ukuran_input

# SKENARIO 3: Menyimpan konfigurasi model
# → Pakai DICTIONARY karena setiap parameter punya nama
konfigurasi = {
    "model"         : "ResNet50",
    "learning_rate" : 0.001,
    "batch_size"    : 32,
    "epochs"        : 100,
    "optimizer"     : "Adam"
}

# SKENARIO 4: Daftar label kelas yang valid dalam dataset
# → Pakai SET karena hanya perlu cek keanggotaan, tidak ada duplikat
label_valid  = {"kucing", "anjing", "burung", "ikan"}
label_prediksi = "kucing"
if label_prediksi in label_valid:
    print("Label valid!")

Struktur Data Bersarang

Dalam praktik nyata, kamu sangat sering menggabungkan struktur data ini satu sama lain — list berisi dictionary, dictionary berisi list, dan seterusnya. Ini disebut struktur data bersarang *(nested)* dan sangat umum dipakai untuk merepresentasikan data yang kompleks seperti dataset atau respons API:

python — struktur data bersarang 
# Dataset sederhana: list of dictionaries
# Ini persis seperti format data yang akan kamu temui di proyek nyata
dataset = [
    {"nama": "Andi",  "usia": 25, "gaji": 8000000,  "lulus": True},
    {"nama": "Budi",  "usia": 30, "gaji": 12000000, "lulus": True},
    {"nama": "Citra", "usia": 22, "gaji": 6000000,  "lulus": False},
]

# Akses data bersarang
print(dataset[0]["nama"])   # → Andi
print(dataset[1]["gaji"])   # → 12000000

# Iterasi untuk mengolah setiap baris data
for orang in dataset:
    status = "✅" if orang["lulus"] else "❌"
    print(f"{status} {orang['nama']:8} | Usia: {orang['usia']} | Gaji: {orang['gaji']:,}")
# → ✅ Andi     | Usia: 25 | Gaji: 8,000,000
# → ✅ Budi     | Usia: 30 | Gaji: 12,000,000
# → ❌ Citra    | Usia: 22 | Gaji: 6,000,000

# Ambil semua nama dari dataset — list comprehension (preview Bab 5)
semua_nama = [orang["nama"] for orang in dataset]
print(semua_nama)  # → ['Andi', 'Budi', 'Citra']

# Hitung rata-rata gaji
semua_gaji  = [orang["gaji"] for orang in dataset]
rata_gaji   = sum(semua_gaji) / len(semua_gaji)
print(f"Rata-rata gaji: Rp {rata_gaji:,.0f}")
# → Rata-rata gaji: Rp 8,666,667
✅

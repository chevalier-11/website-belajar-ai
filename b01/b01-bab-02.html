2.1 Instalasi Python dan Memilih Versi yang Tepat

Python bisa diunduh langsung dari situs resminya di python.org. Saat kamu membuka halaman unduhan, kamu akan melihat ada banyak versi tersedia — mulai dari yang paling lama sampai yang paling baru. Ini bisa membingungkan kalau kamu tidak tahu mana yang harus dipilih.

Aturan sederhananya begini: jangan pakai versi paling baru yang baru rilis, dan jangan pakai versi yang terlalu lama. Versi yang baru rilis sering masih punya masalah yang belum terselesaikan, dan banyak pustaka *(library)* populer belum sempat menyesuaikan diri. Versi yang terlalu lama — misalnya Python 2.x — sudah tidak didukung sama sekali dan tidak boleh dipakai untuk proyek baru.

Yang paling aman adalah memilih versi stabil terbaru dari cabang yang sudah teruji — biasanya ini adalah versi dengan angka minor satu atau dua di bawah yang paling baru. Misalnya, jika versi terbaru adalah 3.13, maka 3.11 atau 3.12 adalah pilihan yang sangat solid. Sebagian besar pustaka kecerdasan buatan dan pembelajaran mesin sudah diuji dengan baik pada versi-versi ini.

💡
Versi Mana yang Aman?

Saat seri ini ditulis, Python 3.11 dan 3.12 adalah pilihan paling stabil untuk proyek kecerdasan buatan. Cek situs python.org untuk versi terkini yang direkomendasikan, dan perhatikan label "stable" di daftar unduhan.

Cara Instalasi di Windows

Unduh installer dari python.org dan jalankan. Ada satu langkah yang sangat penting dan sering dilewatkan: pada layar pertama installer, centang kotak yang bertuliskan "Add Python to PATH" sebelum kamu mengklik Install Now. Kalau kamu melewatkan ini, Python tidak akan bisa dipanggil dari terminal, dan kamu harus menginstal ulang dari awal.

Setelah instalasi selesai, buka terminal (tekan tombol Windows, ketik "cmd" atau "PowerShell", lalu tekan Enter) dan ketik perintah berikut untuk memverifikasi bahwa Python berhasil terinstal:

terminal 
python --version
# Output yang diharapkan: Python 3.11.x atau 3.12.x

pip --version
# Output yang diharapkan: pip 23.x.x from ... (python 3.11)

Kalau kedua perintah di atas menghasilkan output seperti yang diharapkan, selamat — Python sudah terinstal dengan benar. Kalau muncul pesan seperti 'python' is not recognized as an internal or external command, berarti Python belum masuk ke PATH dan kamu perlu menginstal ulang dengan mencentang opsi "Add Python to PATH" tadi.

Cara Instalasi di macOS

macOS biasanya sudah punya Python bawaan, tapi versinya sering kali sudah tua dan tidak disarankan untuk pengembangan. Cara terbaik adalah menginstal Python baru menggunakan Homebrew — sebuah manajer paket *(package manager)* untuk macOS yang sangat populer di kalangan pengembang.

Pertama, instal Homebrew dengan membuka Terminal (bisa dicari di Spotlight dengan menekan Cmd + Spasi) dan menjalankan perintah berikut:

terminal 
# Instal Homebrew (copy dan jalankan seluruh baris ini)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Setelah Homebrew terinstal, instal Python
brew install python@3.12

# Verifikasi instalasi
python3 --version
# Output: Python 3.12.x

Perhatikan bahwa di macOS, kamu biasanya perlu mengetik python3 bukan python untuk memanggil Python 3. Ini adalah perbedaan kecil tapi penting yang sering membuat bingung pemula.

Cara Instalasi di Linux

Di sebagian besar distribusi Linux — terutama Ubuntu dan Debian — Python 3 biasanya sudah terinstal secara bawaan. Kamu bisa memverifikasinya dengan membuka terminal dan menjalankan python3 --version. Kalau belum ada atau versinya terlalu lama, gunakan manajer paket bawaan distribusimu:

terminal (ubuntu/debian) 
# Perbarui daftar paket terlebih dahulu
sudo apt update

# Instal Python 3 dan pip
sudo apt install python3 python3-pip python3-venv

# Verifikasi
python3 --version
pip3 --version
⚠️
Jangan Hapus Python Bawaan di macOS dan Linux

Sistem operasi macOS dan Linux menggunakan Python secara internal untuk berbagai proses sistem. Jangan pernah menghapus atau memodifikasi instalasi Python bawaan sistem. Selalu instal Python versi baru secara terpisah — dengan Homebrew di macOS atau melalui manajer paket di Linux.

2.2 Mengenal Terminal / Baris Perintah

Terminal — yang juga sering disebut baris perintah *(command line)*, konsol *(console)*, atau cangkang *(shell)* — adalah antarmuka berbasis teks untuk berkomunikasi langsung dengan sistem operasi komputer kamu. Alih-alih mengklik ikon dan menu, kamu mengetikkan perintah dan komputer menjalankannya.

Banyak orang yang baru belajar pemrograman menganggap terminal menakutkan karena tampilannya gelap dan isinya hanya teks. Tapi setelah kamu terbiasa, terminal akan terasa jauh lebih cepat dan lebih kuat dari antarmuka grafis untuk banyak tugas pemrograman. Hampir semua pekerjaan sebagai insinyur kecerdasan buatan melibatkan terminal — mulai dari menjalankan skrip, menginstal paket, mengelola kode dengan Git, sampai mendeploy aplikasi ke server.

🖥️
Cara Membuka Terminal

Windows: Tekan Windows + R, ketik "cmd" atau "powershell", tekan Enter. Atau cari "Terminal" di Start Menu. — macOS: Tekan Cmd + Spasi, ketik "Terminal", tekan Enter. Atau buka dari Applications > Utilities > Terminal. — Linux: Tekan Ctrl + Alt + T, atau cari "Terminal" di menu aplikasi.

Perintah Dasar yang Wajib Kamu Kuasai

Kamu tidak perlu menghafal ratusan perintah terminal — cukup kuasai sekitar 10–15 perintah yang paling sering dipakai, dan kamu sudah bisa menangani hampir semua kebutuhan sehari-hari. Berikut ini perintah-perintah paling penting yang perlu kamu tahu:

Perintah (Mac/Linux) Perintah (Windows) Fungsi
pwd cd Tampilkan lokasi folder saat ini
ls dir Tampilkan isi folder saat ini
cd nama-folder cd nama-folder Masuk ke dalam folder
cd .. cd .. Naik satu level ke folder induk
mkdir nama-folder mkdir nama-folder Buat folder baru
touch nama-file.py echo. > nama-file.py Buat file baru yang kosong
rm nama-file del nama-file Hapus file
clear cls Bersihkan layar terminal

Mari kita coba langsung. Buka terminal kamu sekarang dan ikuti urutan perintah berikut ini — ini akan membuatmu membangun rasa percaya diri dengan terminal sejak dini:

terminal — latihan navigasi 
# 1. Lihat di mana kamu sekarang
pwd
# Contoh output: /Users/nama-kamu atau C:\Users\nama-kamu

# 2. Lihat isi folder saat ini
ls        # Mac/Linux
dir       # Windows

# 3. Buat folder baru untuk proyek belajar
mkdir belajar-python

# 4. Masuk ke folder tersebut
cd belajar-python

# 5. Buat file Python pertama
touch halo.py      # Mac/Linux
echo. > halo.py    # Windows

# 6. Konfirmasi file sudah terbuat
ls
# Output: halo.py

Tips Menggunakan Terminal Lebih Efisien

Ada beberapa trik yang akan sangat mempercepat kerja kamu di terminal. Pertama, gunakan tombol Tab untuk melengkapi nama file atau folder secara otomatis — ketik beberapa huruf pertama, tekan Tab, dan terminal akan melengkapinya. Kalau ada lebih dari satu kemungkinan, tekan Tab dua kali untuk melihat semua pilihannya.

Kedua, tekan tombol ↑ (panah atas) di keyboard untuk mengulang perintah terakhir yang kamu ketik. Ini sangat berguna saat kamu ingin menjalankan perintah yang sama berulang kali, atau saat kamu ingin mengedit sedikit perintah sebelumnya tanpa harus mengetik ulang semuanya.

✅
Shortcut Terminal Paling Berguna

Tab = lengkapi otomatis nama file/folder. Panah ↑/↓ = navigasi riwayat perintah. Ctrl + C = hentikan proses yang sedang berjalan (ini sangat penting — gunakan ini ketika program kamu "macet" atau loop tanpa henti). Ctrl + L = bersihkan layar (alternatif dari perintah clear/cls).

2.3 Instalasi dan Konfigurasi VS Code

Editor kode adalah tempat kamu akan menghabiskan sebagian besar waktumu sebagai pengembang. Ada banyak pilihan editor di luar sana — Vim, Emacs, PyCharm, Sublime Text, dan masih banyak lagi. Tapi untuk belajar dan untuk mayoritas pekerjaan industri kecerdasan buatan, Visual Studio Code (biasa disebut VS Code) adalah pilihan terbaik saat ini.

VS Code dibuat oleh Microsoft, gratis, dan lintas platform — bisa dipakai di Windows, macOS, dan Linux. Yang menjadikannya luar biasa bukan fitur bawaannya yang memang sudah bagus, melainkan ekosistem ekstensinya yang sangat kaya. Ada ribuan ekstensi yang dibuat oleh komunitas untuk memperluas kemampuan VS Code — mulai dari penanda sintaksis *(syntax highlighting)* untuk ratusan bahasa pemrograman, integrasi Git, debugger, sampai asisten kode berbasis kecerdasan buatan.

Cara Menginstal VS Code

Unduh VS Code dari code.visualstudio.com dan ikuti proses instalasi standar sesuai sistem operasimu. Proses ini biasanya hanya butuh beberapa menit. Setelah terinstal, buka VS Code dan kamu akan disambut dengan layar selamat datang.

Ekstensi Wajib untuk Python

Langkah pertama setelah menginstal VS Code adalah memasang ekstensi Python. Ekstensi ini akan memberikanmu fitur-fitur penting seperti pelengkap kode otomatis *(autocomplete)*, penanda kesalahan secara langsung saat kamu mengetik, kemampuan menjalankan dan men-debug kode Python, serta integrasi dengan Jupyter Notebook.

Cara memasang ekstensi: klik ikon yang terlihat seperti empat kotak kecil di bilah sisi kiri VS Code (atau tekan Ctrl+Shift+X), lalu cari dan instal ekstensi-ekstensi berikut:

  • Python (oleh Microsoft) — ini yang paling penting, harus ada
  • Pylance (oleh Microsoft) — memberikan pengecekan tipe data dan pelengkap kode yang lebih cerdas
  • Jupyter (oleh Microsoft) — untuk menjalankan Jupyter Notebook langsung di dalam VS Code
  • GitLens (oleh GitKraken) — membuat Git jauh lebih mudah digunakan dari dalam editor

Konfigurasi Dasar yang Direkomendasikan

Ada beberapa pengaturan VS Code yang sangat disarankan untuk diaktifkan. Buka pengaturan dengan menekan Ctrl+, (tanda koma), lalu cari dan aktifkan pengaturan berikut:

  • Format On Save — otomatis merapikan format kode setiap kali kamu menyimpan file. Ini membiasakanmu menulis kode yang rapi sejak awal.
  • Auto Save — simpan file otomatis setelah beberapa detik tidak ada perubahan. Ini mencegah kamu kehilangan pekerjaan karena lupa menyimpan.
  • Word Wrap — membuat teks yang terlalu panjang otomatis turun ke baris berikutnya tanpa kamu harus scroll ke kanan.

Untuk mengatur formatter Python, VS Code biasanya akan menanyakan mana yang ingin kamu gunakan saat pertama kali kamu membuka file Python. Pilih Black — ini adalah formatter Python yang paling populer di industri dan hampir tidak memerlukan konfigurasi tambahan.

terminal — instal black formatter 
# Instal Black, formatter Python standar industri
pip install black

# Coba format sebuah file (setelah kamu punya file Python)
black nama-file.py

# Black akan otomatis merapikan kode kamu sesuai standar PEP 8

Menjalankan Kode Python Pertamamu di VS Code

Setelah semuanya terpasang, mari coba jalankan kode Python pertama di VS Code. Buat file baru dengan nama halo_dunia.py, tulis kode berikut, lalu simpan:

python — halo_dunia.py 
# Program Python pertama
print("Halo, dunia!")
print("Saya sedang belajar Python untuk menjadi Insinyur KA.")

# Coba operasi sederhana
angka_satu = 10
angka_dua  = 25
jumlah = angka_satu + angka_dua
print(f"Hasil penjumlahan: {jumlah}")

Untuk menjalankannya, kamu bisa: (1) klik tombol segitiga hijau ▶ di pojok kanan atas editor, atau (2) klik kanan di area kode dan pilih "Run Python File in Terminal", atau (3) buka terminal terintegrasi VS Code dengan Ctrl+` (tanda backtick) dan ketik python halo_dunia.py. Cara ketiga adalah cara yang paling sering dipakai oleh insinyur profesional.

2.4 Mengenal dan Menyiapkan Lingkungan Virtual

Ini adalah salah satu konsep yang paling sering dilewatkan oleh pemula, tapi sangat penting untuk dipahami sejak awal. Lingkungan virtual *(virtual environment)* adalah cara untuk mengisolasi paket *(package)* Python yang dipakai oleh setiap proyek secara terpisah satu sama lain.

Bayangkan kamu sedang mengerjakan dua proyek berbeda. Proyek A membutuhkan pustaka numpy versi 1.23, tapi Proyek B membutuhkan versi 1.26 yang sudah punya fitur baru. Tanpa lingkungan virtual, kamu tidak bisa punya kedua versi secara bersamaan di satu komputer — menginstal versi baru akan menimpa yang lama dan bisa merusak Proyek A. Dengan lingkungan virtual, setiap proyek punya "ruangan" sendiri yang berisi paket-paket versi yang mereka butuhkan, tanpa saling mengganggu.

📦
Aturan Emas: Satu Proyek, Satu Lingkungan Virtual

Selalu buat lingkungan virtual baru untuk setiap proyek Python yang kamu mulai. Ini adalah praktik standar industri yang akan menyelamatkanmu dari konflik versi paket yang bisa sangat sulit di-debug.

Membuat dan Mengaktifkan Lingkungan Virtual

Python sudah menyertakan modul venv secara bawaan untuk membuat lingkungan virtual — kamu tidak perlu menginstal apa-apa tambahan. Berikut cara penggunaannya:

terminal — membuat lingkungan virtual 
# 1. Masuk ke folder proyek kamu
cd belajar-python

# 2. Buat lingkungan virtual baru dengan nama "venv"
python -m venv venv       # Windows
python3 -m venv venv      # macOS/Linux

# 3. Aktifkan lingkungan virtual
venv\Scripts\activate     # Windows (Command Prompt)
venv\Scripts\Activate.ps1 # Windows (PowerShell)
source venv/bin/activate  # macOS/Linux

# 4. Setelah aktif, nama lingkungan akan muncul di awal baris terminal
# Contoh: (venv) C:\Users\kamu\belajar-python>
# Atau:   (venv) nama-kamu@komputer:~/belajar-python$

# 5. Sekarang instal paket — hanya tersimpan di lingkungan ini
pip install numpy pandas

# 6. Untuk keluar dari lingkungan virtual
deactivate

Perhatikan tanda (venv) di awal baris terminal. Ini adalah penanda bahwa kamu sedang berada di dalam lingkungan virtual aktif. Setiap paket yang kamu instal dengan pip install selama tanda ini muncul akan tersimpan di dalam folder venv di proyekmu, bukan di instalasi Python utama di komputermu.

Menyimpan Daftar Paket yang Digunakan

Salah satu manfaat besar lingkungan virtual adalah kamu bisa dengan mudah membuat daftar semua paket yang dipakai proyekmu, sehingga orang lain (atau kamu sendiri di komputer lain) bisa menginstal semua yang dibutuhkan sekaligus. File daftar ini secara konvensi bernama requirements.txt:

terminal — requirements.txt 
# Simpan semua paket yang terinstal ke dalam file
pip freeze > requirements.txt

# Untuk menginstal dari file ini (di komputer lain atau lingkungan baru)
pip install -r requirements.txt

# Lihat isi requirements.txt — contohnya akan terlihat seperti ini:
# numpy==1.26.4
# pandas==2.1.4
# scikit-learn==1.3.2
⚠️
Jangan Upload Folder venv ke GitHub

Folder venv berisi ribuan file biner yang ukurannya besar dan tidak perlu disimpan di repositori. Cukup simpan file requirements.txt saja. Di bab Git nanti, kamu akan belajar cara menambahkan venv/ ke file .gitignore supaya tidak sengaja ter-upload.

Lingkungan Virtual di VS Code

VS Code bisa otomatis mendeteksi lingkungan virtual di proyekmu. Setelah kamu membuat lingkungan virtual, buka folder proyekmu di VS Code (File → Open Folder), lalu buka file Python apapun. Di pojok kanan bawah VS Code, kamu akan melihat nama dan versi Python yang sedang aktif — klik di sana untuk memilih interpreter Python dari lingkungan virtualmu. Setelah dipilih, terminal terintegrasi VS Code akan otomatis mengaktifkan lingkungan virtual tersebut setiap kali dibuka.

2.5 Jupyter Notebook: Kapan Pakai, Kapan Tidak

Jupyter Notebook adalah lingkungan pengembangan interaktif yang memungkinkan kamu menulis dan menjalankan kode Python dalam potongan-potongan kecil yang disebut sel *(cell)*, satu per satu, sambil langsung melihat hasilnya di bawah setiap sel. Kamu juga bisa menyelipkan teks penjelasan, rumus matematika, dan visualisasi grafik di antara sel-sel kode — semuanya dalam satu dokumen yang bisa dibagikan.

Jupyter sangat populer di kalangan ilmuwan data *(data scientist)* dan peneliti karena format ini memudahkan eksplorasi data secara interaktif. Kamu bisa mencoba sesuatu, melihat hasilnya langsung, mengubah kodenya, dan mencoba lagi — tanpa harus menjalankan ulang seluruh program setiap kali.

Kapan Pakai Jupyter Notebook

Jupyter sangat cocok digunakan untuk beberapa situasi spesifik: eksplorasi data awal — ketika kamu baru pertama kali melihat sebuah dataset dan ingin memahami strukturnya, analisis dan visualisasi — ketika kamu ingin membuat grafik dan melihat hasilnya langsung, pembelajaran dan eksperimen — mencoba konsep baru dengan cepat tanpa harus membuat file .py yang lengkap, dan presentasi hasil — Jupyter Notebook bisa diekspor sebagai HTML atau PDF yang terlihat rapi untuk dibagikan ke orang lain.

Kapan Tidak Pakai Jupyter Notebook

Jupyter bukan pilihan yang tepat untuk semua situasi. Pengembangan aplikasi — ketika kamu membangun program atau API yang nyata, kamu butuh file .py yang terstruktur, bukan notebook. Kode yang akan berjalan di produksi — sistem di server tidak mengenal format .ipynb (format file Jupyter). Proyek kolaborasi besar — kode di notebook lebih sulit di-review dan di-debug karena tidak punya struktur yang jelas.

📌
Rekomendasi Penggunaan di Seri Ini

Dalam seri ini, kamu akan banyak menggunakan Jupyter di Buku 2 (analisis data) dan Buku 3 (eksperimen model ML). Tapi di Buku 4 dan 5, kamu akan beralih ke file .py murni karena kita sudah mulai membangun sistem yang lebih kompleks dan akan dideploy. Ini mencerminkan cara kerja yang sesungguhnya di industri.

Cara Instal dan Menjalankan Jupyter

Menginstal Jupyter sangat mudah — cukup dengan satu perintah pip. Pastikan lingkungan virtualmu sudah aktif sebelum menginstal:

terminal 
# Pastikan lingkungan virtual aktif dulu (ada tanda (venv) di terminal)

# Instal JupyterLab — versi modern dari Jupyter Notebook
pip install jupyterlab

# Jalankan JupyterLab
jupyter lab

# Terminal akan menampilkan URL seperti ini:
# http://localhost:8888/lab?token=xxxxxxxxxxxx
# Peramban akan otomatis terbuka, atau kamu bisa copy-paste URL-nya

# Untuk menghentikan server Jupyter, tekan Ctrl+C di terminal

Setelah JupyterLab terbuka di peramban, kamu bisa membuat notebook baru dengan mengklik tombol "+ New" dan memilih "Python 3". Sebuah notebook baru akan terbuka dengan satu sel kosong siap untuk diisi kode.

Di dalam notebook, ada beberapa pintasan keyboard yang wajib kamu tahu: Shift + Enter untuk menjalankan sel saat ini dan pindah ke sel berikutnya, Ctrl + Enter untuk menjalankan sel saat ini tanpa pindah, A untuk menambah sel baru di atas, B untuk menambah sel baru di bawah, dan D + D (tekan D dua kali) untuk menghapus sel yang dipilih. Biasakan pintasan-pintasan ini dan produktivitasmu di Jupyter akan meningkat drastis.

jupyter notebook — contoh sel pertama 
# Coba ini di sel pertama notebook kamu
print("Halo dari Jupyter!")

nama = "Calon Insinyur KA"
angka = [1, 2, 3, 4, 5]
rata_rata = sum(angka) / len(angka)

print(f"Nama: {nama}")
print(f"Rata-rata angka: {rata_rata}")

# Jupyter otomatis menampilkan nilai ekspresi terakhir di sel
# tanpa perlu print() — ini fitur khas Jupyter
rata_rata

Tekan Shift + Enter untuk menjalankan sel di atas. Kamu akan melihat output langsung muncul di bawahnya — itulah keunggulan utama Jupyter. Selamat, lingkungan kerjamu sudah siap sepenuhnya!

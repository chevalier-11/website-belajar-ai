7.1 Membaca dan Menulis File Teks

Hampir semua pekerjaan data dimulai dari file — entah itu file teks berisi log sistem, file CSV berisi data penjualan, atau file JSON dari API. Python punya cara yang sangat bersih untuk membuka, membaca, menulis, dan menutup file menggunakan fungsi bawaan open().

Saat membuka file, kamu perlu menentukan mode — apakah file dibuka untuk dibaca, ditulis, atau ditambahi. Selain mode, penting juga menentukan encoding — cara karakter dikodekan dalam file. Untuk teks bahasa Indonesia yang mengandung karakter seperti "é", "ñ", atau bahkan emoji, selalu gunakan encoding="utf-8" untuk menghindari error karakter aneh.

python 
# Mode file yang paling sering dipakai:
# "r"  → read   (baca, file harus ada)        ← default
# "w"  → write  (tulis, buat baru / timpa lama)
# "a"  → append (tambah ke akhir file yang ada)
# "r+" → read+write (baca dan tulis)

# Cara lama — harus ingat menutup file sendiri
f = open("catatan.txt", "r", encoding="utf-8")
isi = f.read()
f.close()   # mudah terlupakan → file bisa bocor (resource leak)

# Cara modern — pakai 'with' statement (context manager)
# File OTOMATIS ditutup saat keluar dari blok 'with', bahkan kalau error
with open("catatan.txt", "r", encoding="utf-8") as f:
    isi = f.read()   # baca seluruh file sebagai satu string
print(isi)
# setelah blok with, f sudah ditutup secara otomatis
💡
Selalu Pakai with Statement

Pernyataan with open(...) as f: adalah cara yang benar dan dianjurkan untuk membuka file di Python. Ia memastikan file selalu ditutup — bahkan jika terjadi error di dalam bloknya. Lupa menutup file bisa menyebabkan data tidak tersimpan dengan benar atau file terkunci sehingga program lain tidak bisa mengaksesnya. Jadikan ini kebiasaan dari awal.

Tiga Cara Membaca File Teks

Ada tiga metode untuk membaca isi file, masing-masing cocok untuk situasi yang berbeda. read() membaca seluruh file sekaligus sebagai satu string — cocok untuk file kecil. readlines() membaca seluruh file dan mengembalikan list dimana setiap elemen adalah satu baris. readline() membaca satu baris saja setiap kali dipanggil. Untuk file yang sangat besar, cara paling hemat memori adalah mengiterasi file langsung dengan for:

python 
# Buat file contoh dulu untuk demo
with open("siswa.txt", "w", encoding="utf-8") as f:
    f.write("Andi,85\n")
    f.write("Budi,92\n")
    f.write("Citra,78\n")
    f.write("Dian,95\n")

# 1. read() — seluruh isi sebagai satu string
with open("siswa.txt", encoding="utf-8") as f:
    isi = f.read()
print(repr(isi))
# → 'Andi,85\nBudi,92\nCitra,78\nDian,95\n'

# 2. readlines() — list per baris
with open("siswa.txt", encoding="utf-8") as f:
    baris_baris = f.readlines()
print(baris_baris)
# → ['Andi,85\n', 'Budi,92\n', 'Citra,78\n', 'Dian,95\n']

# 3. Iterasi langsung — paling hemat memori, cocok file besar
with open("siswa.txt", encoding="utf-8") as f:
    for baris in f:
        baris = baris.strip()   # hapus \n di akhir
        nama, nilai = baris.split(",")
        print(f"{nama}: {nilai}")

Menulis dan Menambah Konten ke File

Mode "w" membuat file baru atau menimpa isi file yang sudah ada — hati-hati dengan ini. Mode "a" menambahkan konten ke akhir file yang ada tanpa menghapus isinya, cocok untuk menulis log atau mencatat hasil eksperimen secara bertahap. Untuk menulis banyak baris sekaligus, gunakan writelines() atau gabungkan dengan join():

python 
# Menulis — mode "w" timpa file lama
hasil_eksperimen = [
    "Eksperimen 1: lr=0.001, akurasi=0.82\n",
    "Eksperimen 2: lr=0.01,  akurasi=0.87\n",
    "Eksperimen 3: lr=0.0001, akurasi=0.79\n",
]

with open("log_eksperimen.txt", "w", encoding="utf-8") as f:
    f.writelines(hasil_eksperimen)

# Menambah — mode "a" tambah ke akhir, tidak timpa
with open("log_eksperimen.txt", "a", encoding="utf-8") as f:
    f.write("Eksperimen 4: lr=0.005, akurasi=0.89\n")

# Menulis dengan f-string — cara yang paling sering dipakai
import datetime
waktu = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")

with open("log_pelatihan.txt", "a", encoding="utf-8") as f:
    f.write(f"[{waktu}] epoch=50, loss=0.0234, val_acc=0.912\n")

print("Log tersimpan!")
⚠️
Mode "w" Menghapus File Lama Tanpa Peringatan

Membuka file dengan mode "w" langsung mengosongkan isi file tersebut, bahkan sebelum kamu menulis apapun. Tidak ada dialog konfirmasi, tidak ada tong sampah — data langsung hilang. Kalau niatmu menambah konten ke file yang sudah ada, gunakan mode "a". Kalau memang ingin membuat file baru tapi file sudah ada, pertimbangkan untuk cek dulu dengan os.path.exists() sebelum menulis.

7.2 Bekerja dengan File CSV

CSV *(Comma-Separated Values)* adalah format file yang paling umum untuk data terstruktur — hampir semua dataset yang kamu unduh dari Kaggle, pemerintah, atau portal data publik tersedia dalam format ini. Setiap baris adalah satu rekaman data, dan setiap kolom dipisahkan oleh koma (atau karakter pemisah lain seperti titik koma atau tab).

Python punya modul bawaan bernama csv yang sudah menangani semua kerumitan format CSV — termasuk nilai yang mengandung koma di dalamnya, tanda kutip, dan baris kosong. Di bab-bab selanjutnya kamu akan memakai Pandas yang jauh lebih canggih untuk pekerjaan data serius, tapi memahami modul csv bawaan tetap penting karena ia tersedia di semua lingkungan tanpa perlu instalasi.

python 
import csv

# Buat file CSV contoh dulu
header = ["nama", "nilai", "kelas"]
data   = [
    ["Andi",    85, "XII-A"],
    ["Budi",    92, "XII-A"],
    ["Citra",   78, "XII-B"],
    ["Dian",    95, "XII-B"],
    ["Evan",    88, "XII-A"],
]

# Menulis file CSV
with open("siswa.csv", "w", newline="", encoding="utf-8") as f:
    penulis = csv.writer(f)
    penulis.writerow(header)   # tulis baris header
    penulis.writerows(data)    # tulis semua baris data sekaligus

print("File siswa.csv berhasil dibuat!")
💡
newline="" Saat Menulis CSV di Windows

Saat membuka file untuk menulis CSV, selalu tambahkan argumen newline="". Ini mencegah modul csv dan Python menambahkan karakter baris baru yang berbeda secara bersamaan, yang akan menghasilkan baris kosong ekstra di antara setiap baris data di file (khususnya di Windows). Ini quirk kecil Python yang kalau dilewatkan bisa bikin bingung saat file dibuka di Excel.

Membaca CSV sebagai List dan Dictionary

Ada dua pembaca CSV utama yang perlu kamu tahu: csv.reader yang membaca setiap baris sebagai list biasa, dan csv.DictReader yang membaca setiap baris sebagai dictionary dengan kunci dari baris header. Untuk data dengan banyak kolom, DictReader jauh lebih nyaman karena kamu bisa mengakses kolom berdasarkan namanya, bukan posisi angkanya:

python 
# Membaca CSV sebagai list — csv.reader
with open("siswa.csv", encoding="utf-8") as f:
    pembaca = csv.reader(f)
    header  = next(pembaca)   # ambil baris pertama sebagai header
    print(f"Kolom: {header}")
    for baris in pembaca:
        print(baris)
# → Kolom: ['nama', 'nilai', 'kelas']
# → ['Andi', '85', 'XII-A']
# → ['Budi', '92', 'XII-A']  ← nilai adalah STRING, bukan int!

# Membaca CSV sebagai dict — csv.DictReader (lebih nyaman)
with open("siswa.csv", encoding="utf-8") as f:
    pembaca = csv.DictReader(f)   # header otomatis jadi kunci dict
    siswa_list = list(pembaca)

# Akses berdasarkan nama kolom, bukan indeks angka
for siswa in siswa_list:
    nama  = siswa["nama"]
    nilai = int(siswa["nilai"])   # konversi dari string ke int
    kelas = siswa["kelas"]
    print(f"{nama} ({kelas}): {nilai}")

Contoh Nyata: Analisis Data dari CSV

Setelah membaca data dari CSV, kamu bisa langsung memproses dan menganalisisnya dengan semua yang sudah kamu pelajari di bab-bab sebelumnya. Ini adalah alur kerja yang akan terus kamu ulangi sepanjang karir di data science — baca data, olah, hasilkan insight:

python — analisis lengkap dari CSV 
import csv

def analisis_nilai_siswa(path_file):
    """Baca file CSV siswa dan hasilkan laporan ringkas."""
    with open(path_file, encoding="utf-8") as f:
        data = list(csv.DictReader(f))

    # Konversi nilai ke integer
    for baris in data:
        baris["nilai"] = int(baris["nilai"])

    semua_nilai = [b["nilai"] for b in data]

    # Hitung statistik
    rata    = sum(semua_nilai) / len(semua_nilai)
    tertinggi = max(semua_nilai)
    terendah  = min(semua_nilai)
    lulus     = [b for b in data if b["nilai"] >= 75]

    # Kelompokkan per kelas
    per_kelas = {}
    for b in data:
        k = b["kelas"]
        if k not in per_kelas:
            per_kelas[k] = []
        per_kelas[k].append(b["nilai"])

    # Cetak laporan
    print("=== Laporan Nilai Siswa ===")
    print(f"Total siswa  : {len(data)}")
    print(f"Rata-rata    : {rata:.1f}")
    print(f"Tertinggi    : {tertinggi}")
    print(f"Terendah     : {terendah}")
    print(f"Lulus (≥75)  : {len(lulus)} dari {len(data)}")
    print()
    print("Rata-rata per kelas:")
    for kelas, nilai_kelas in sorted(per_kelas.items()):
        r = sum(nilai_kelas) / len(nilai_kelas)
        print(f"  {kelas}: {r:.1f}")

analisis_nilai_siswa("siswa.csv")

7.3 Memahami dan Membaca Pesan Error

Error adalah bagian normal dari pemrograman — bukan tanda bahwa kamu tidak berbakat. Bahkan programmer paling berpengalaman pun menghabiskan banyak waktu membaca pesan error. Yang membedakan pemula dan yang sudah berpengalaman bukan frekuensi error, melainkan kecepatan membacanya dan kemampuan menemukan penyebabnya.

Saat program Python berhenti karena kesalahan, ia mencetak traceback — jejak perjalanan error dari titik awal pemanggilan sampai baris yang sebenarnya bermasalah. Membaca traceback dari bawah ke atas adalah kuncinya: baris paling bawah adalah masalah yang sebenarnya, baris-baris di atasnya adalah konteks pemanggilan yang mengantarkan ke sana.

python — contoh traceback 
# Contoh kode yang menghasilkan error
def bagi(a, b):
    return a / b

def hitung_rata(nilai_list):
    total = sum(nilai_list)
    n     = len(nilai_list)
    return bagi(total, n)   # memanggil bagi()

hitung_rata([])   # list kosong → n = 0 → error!

# Error yang muncul di konsol:
# Traceback (most recent call last):
#   File "belajar.py", line 9, in <module>
#     hitung_rata([])                   ← titik panggilan pertama
#   File "belajar.py", line 7, in hitung_rata
#     return bagi(total, n)             ← lanjut ke sini
#   File "belajar.py", line 2, in bagi
#     return a / b                      ← INI yang bermasalah
# ZeroDivisionError: division by zero   ← nama + pesan error
#                                          ↑ baca dari sini dulu!

Python punya sistem hirarki error yang sangat terorganisir. Setiap jenis error punya nama yang deskriptif — begitu kamu melihat nama errornya, kamu sudah punya gambaran tentang apa yang salah. Berikut error yang hampir pasti kamu temui berulang kali:

Nama Error Artinya Contoh Penyebab

Komunitas Stack Overflow hampir pasti sudah menjawab error Python yang kamu temui, karena Python sudah dipakai jutaan orang selama puluhan tahun.

7.4 try, except, dan finally

Mengetahui jenis-jenis error sudah bagus — tapi lebih bagus lagi kalau programmu bisa menanganinya sendiri tanpa berhenti. Inilah tujuan penanganan pengecualian *(exception handling)* menggunakan blok try-except. Kamu menaruh kode yang mungkin gagal di dalam blok try, dan menentukan apa yang harus dilakukan kalau error terjadi di dalam blok except.

Filosofi yang perlu kamu pegang: gunakan try-except untuk menangani error yang bisa kamu antisipasi dan pulihkan — bukan untuk menyembunyikan semua error. Blok except yang kosong atau yang menangkap Exception terlalu luas bisa menyembunyikan bug nyata dan membuat debugging menjadi mimpi buruk.

python 
# Struktur dasar try-except
try:
    angka = int(input("Masukkan angka: "))
    hasil = 100 / angka
    print(f"100 / {angka} = {hasil}")
except ValueError:
    print("Itu bukan angka. Masukkan bilangan bulat.")
except ZeroDivisionError:
    print("Tidak bisa dibagi dengan nol.")

# Menangkap beberapa error sekaligus dalam satu except
try:
    nilai = int(input("Nilai: "))
except (ValueError, OverflowError) as e:
    print(f"Input tidak valid: {e}")

# Blok 'else' — dijalankan jika TIDAK ada error
try:
    f = open("data.txt", encoding="utf-8")
except FileNotFoundError:
    print("File tidak ditemukan.")
else:
    isi = f.read()    # hanya jalan kalau open() berhasil
    f.close()
    print(f"Berhasil membaca {len(isi)} karakter.")

finally — Selalu Dijalankan

Blok finally dijalankan selalu — baik ada error maupun tidak. Ini berguna untuk kode pembersihan *(cleanup)* yang harus berjalan apapun yang terjadi, seperti menutup file, menutup koneksi database, atau melepaskan sumber daya lain. Dalam praktiknya, blok with yang sudah kamu pelajari menangani ini secara otomatis untuk file, tapi finally penting untuk sumber daya lain:

python 
# finally dijalankan SELALU — cocok untuk cleanup
def proses_file(path):
    f = None
    try:
        f = open(path, encoding="utf-8")
        data = f.read()
        print(f"Berhasil: {len(data)} karakter dibaca")
        return data
    except FileNotFoundError:
        print(f"Error: file '{path}' tidak ditemukan")
        return None
    except PermissionError:
        print(f"Error: tidak ada izin membaca '{path}'")
        return None
    finally:
        if f is not None:
            f.close()   # selalu tutup file, apapun yang terjadi
        print("Blok finally: pembersihan selesai.")

proses_file("ada.txt")     # akan cetak "finally" meski berhasil
proses_file("tidak_ada.txt")  # akan cetak "finally" meski error

Mengangkat Error Sendiri dengan raise

Kamu juga bisa sengaja memunculkan error menggunakan raise — berguna untuk memvalidasi input di fungsi dan memberikan pesan error yang informatif kepada pengguna fungsimu. Ini jauh lebih baik daripada membiarkan error membingungkan muncul di tempat yang tidak terduga:

python 
def normalisasi(data, min_val, max_val):
    """Normalisasi list angka ke rentang 0–1."""
    if not data:
        raise ValueError("Data tidak boleh kosong")
    if min_val >= max_val:
        raise ValueError(f"min_val ({min_val}) harus lebih kecil dari max_val ({max_val})")
    rentang = max_val - min_val
    return [(x - min_val) / rentang for x in data]

# Penggunaan normal
print(normalisasi([20, 50, 80], 0, 100))
# → [0.2, 0.5, 0.8]

# Tangkap error yang kita sendiri angkat
try:
    normalisasi([], 0, 100)
except ValueError as e:
    print(f"Error validasi: {e}")
# → Error validasi: Data tidak boleh kosong

try:
    normalisasi([1, 2, 3], 100, 0)
except ValueError as e:
    print(f"Error validasi: {e}")
# → Error validasi: min_val (100) harus lebih kecil dari max_val (0)

7.5 Menulis Kode yang Gagal dengan Anggun

"Gagal dengan anggun" *(fail gracefully)* adalah prinsip penting dalam rekayasa perangkat lunak: ketika sesuatu tidak berjalan sesuai rencana, program harus memberikan informasi yang berguna, membersihkan sumber daya, dan jika memungkinkan, melanjutkan bekerja dengan cara yang masuk akal — bukan langsung crash dan meninggalkan pengguna bingung.

Dalam konteks kecerdasan buatan dan data science, ini sangat relevan: kamu sering memproses ratusan atau ribuan file dan tidak ingin satu file yang rusak menghentikan seluruh proses. Kamu ingin program bisa melewati file bermasalah, mencatatnya, dan terus memproses sisanya.

Pola: Memproses Banyak File dengan Tangguh

python 
import csv
import os

def baca_satu_csv(path):
    """Baca satu file CSV, kembalikan list of dict atau None jika gagal."""
    try:
        with open(path, encoding="utf-8") as f:
            return list(csv.DictReader(f))
    except FileNotFoundError:
        print(f"  [LEWATI] File tidak ditemukan: {path}")
        return None
    except PermissionError:
        print(f"  [LEWATI] Tidak ada izin baca: {path}")
        return None
    except UnicodeDecodeError:
        print(f"  [LEWATI] Encoding tidak sesuai: {path}")
        return None

def proses_semua_csv(daftar_path):
    """Proses semua file CSV, laporkan yang gagal."""
    berhasil = 0
    gagal    = 0
    semua_data = []

    print(f"Memproses {len(daftar_path)} file...")
    for path in daftar_path:
        data = baca_satu_csv(path)
        if data is not None:
            semua_data.extend(data)
            berhasil += 1
        else:
            gagal += 1

    print(f"\nSelesai: {berhasil} berhasil, {gagal} gagal.")
    print(f"Total baris data terkumpul: {len(semua_data)}")
    return semua_data

# Simulasi pemrosesan — dua ada, satu tidak ada
file_list = ["siswa.csv", "guru.csv", "nilai_tidak_ada.csv"]
semua = proses_semua_csv(file_list)

Pola: Mencatat Error ke File Log

Dalam program produksi, error tidak boleh hanya dicetak ke layar — harus dicatat ke file log agar bisa ditelusuri nanti. Ini adalah praktik standar di industri. Modul logging bawaan Python adalah cara yang benar untuk ini, tapi untuk memulai, menulis ke file teks sudah cukup untuk belajar:

python 
import datetime

def catat_log(level, pesan, path_log="app.log"):
    """Catat pesan log ke file dengan timestamp."""
    waktu = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    baris = f"[{waktu}] [{level.upper():8}] {pesan}\n"
    with open(path_log, "a", encoding="utf-8") as f:
        f.write(baris)
    print(baris, end="")   # cetak ke layar sekaligus

def muat_konfigurasi(path):
    """Muat konfigurasi dari file, gunakan default jika gagal."""
    default = {"lr": 0.001, "batch_size": 32, "epochs": 100}
    try:
        with open(path, encoding="utf-8") as f:
            import json
            config = json.load(f)
        catat_log("info", f"Konfigurasi dimuat dari {path}")
        return config
    except FileNotFoundError:
        catat_log("warning", f"File konfigurasi '{path}' tidak ditemukan. Pakai default.")
        return default
    except json.JSONDecodeError as e:
        catat_log("error", f"File konfigurasi '{path}' rusak: {e}. Pakai default.")
        return default

config = muat_konfigurasi("config.json")
print(f"\nKonfigurasi aktif: {config}")
⚠️
Jangan Tangkap Exception Terlalu Luas

Menulis except Exception: atau except: tanpa nama error adalah kebiasaan buruk — ia menyembunyikan semua jenis error, termasuk bug nyata yang seharusnya diperbaiki. Tangkap hanya error yang kamu tahu cara menanganinya dan tangkap sekecil mungkin. Kalau kamu tidak yakin error apa yang mungkin muncul, biarkan error itu muncul dulu, baca, baru tambahkan penanganannya.

Pola Lengkap: Memproses CSV dengan Validasi Penuh

Mari gabungkan semua yang sudah kamu pelajari di bab ini — membaca file, menangani error, dan logging — menjadi satu fungsi yang benar-benar siap pakai di proyek nyata:

python — pola siap pakai 
import csv

def muat_data_nilai(path_csv, kolom_nilai="nilai"):
    """
    Muat data nilai dari CSV dengan validasi dan penanganan error lengkap.

    Parameter:
        path_csv    (str)  : path ke file CSV
        kolom_nilai (str)  : nama kolom nilai numerik

    Kembalian:
        tuple: (list_data_valid, jumlah_dilewati) atau (None, 0) jika file gagal dimuat
    """
    try:
        with open(path_csv, encoding="utf-8") as f:
            baris_mentah = list(csv.DictReader(f))
    except FileNotFoundError:
        print(f"[ERROR] File tidak ditemukan: {path_csv}")
        return None, 0

    data_valid   = []
    dilewati     = 0
    pesan_error  = []

    for i, baris in enumerate(baris_mentah, start=2):  # mulai dari 2 (baris 1 = header)
        if kolom_nilai not in baris:
            pesan_error.append(f"  Baris {i}: kolom '{kolom_nilai}' tidak ada")
            dilewati += 1
            continue
        try:
            baris[kolom_nilai] = float(baris[kolom_nilai])
            data_valid.append(baris)
        except ValueError:
            pesan_error.append(f"  Baris {i}: nilai '{baris[kolom_nilai]}' bukan angka")
            dilewati += 1

    if pesan_error:
        print(f"[WARNING] {dilewati} baris dilewati:")
        for msg in pesan_error:
            print(msg)

    print(f"[OK] {len(data_valid)} baris valid dimuat dari {path_csv}")
    return data_valid, dilewati

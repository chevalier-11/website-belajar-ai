6.1 Mendefinisikan dan Memanggil Fungsi

Fungsi adalah blok kode yang diberi nama dan bisa dijalankan kapan saja hanya dengan menyebut namanya. Prinsip di baliknya sangat sederhana: kamu menulis sekumpulan instruksi sekali, lalu memakainya berulang kali. Ini menghilangkan duplikasi kode — salah satu sumber bug dan rasa frustrasi terbesar dalam pemrograman.

Di Python, fungsi didefinisikan dengan kata kunci def, diikuti nama fungsi, tanda kurung, dan titik dua. Semua kode di dalam fungsi harus diberi indentasi 4 spasi — sama persis dengan aturan indentasi yang sudah kamu pelajari di Bab 5. Mendefinisikan fungsi tidak langsung menjalankannya; fungsi baru berjalan ketika kamu memanggilnya — menyebut namanya diikuti tanda kurung.

python 
# Mendefinisikan fungsi — belum dijalankan
def sapa():
    print("Halo! Selamat belajar Python.")

# Memanggil fungsi — baru dijalankan sekarang
sapa()   # → Halo! Selamat belajar Python.
sapa()   # bisa dipanggil berulang kali
sapa()

# Fungsi bisa memanggil fungsi lain
def mulai_sesi_belajar():
    sapa()
    print("Sesi belajar dimulai. Semangat!")

mulai_sesi_belajar()

Docstring — Dokumentasi di Dalam Fungsi

Setiap fungsi yang kamu buat sebaiknya punya docstring — teks penjelasan yang ditulis tepat di baris pertama setelah def, dibungkus tanda kutip tiga. Docstring bukan komentar biasa; ia adalah bagian dari fungsi yang bisa diakses secara programatik. Ketika kamu bekerja di tim atau kembali ke kode sendiri setelah beberapa bulan, docstring yang baik bisa menyelamatkanmu dari kebingungan panjang:

python 
def hitung_rata_rata(data):
    """
    Hitung rata-rata dari list angka.

    Parameter:
        data (list): list angka yang akan dihitung rata-ratanya

    Kembalian:
        float: nilai rata-rata, atau None jika list kosong
    """
    if not data:
        return None
    return sum(data) / len(data)

# Mengakses docstring fungsi
print(hitung_rata_rata.__doc__)

# Atau gunakan help()
help(hitung_rata_rata)
💡
Prinsip DRY — Don't Repeat Yourself

Prinsip paling fundamental dalam rekayasa perangkat lunak adalah DRY: jangan tulis kode yang sama lebih dari sekali. Kalau kamu mendapati diri menyalin-tempel blok kode yang sama ke dua atau lebih tempat, itu tanda yang sangat jelas bahwa kode itu seharusnya menjadi fungsi. Duplikasi kode adalah sumber bug — kalau ada kesalahan, kamu harus memperbaikinya di semua tempat, dan mudah sekali melewatkan satu.

Kapan Membuat Fungsi?

Ada tiga tanda bahwa sesuatu seharusnya jadi fungsi. Pertama, kamu menulis blok kode yang sama (atau sangat mirip) lebih dari sekali. Kedua, sebuah bagian kode melakukan satu tugas yang bisa diberi nama yang jelas — seperti "hitung_akurasi" atau "bersihkan_teks". Ketiga, kamu ingin bisa menguji bagian kode itu secara terpisah dari bagian lainnya. Fungsi yang baik melakukan satu hal dengan baik, bukan banyak hal sekaligus.

python — sebelum vs sesudah refactoring 
# ── SEBELUM: kode berulang, susah diubah ──────────────────
nilai_kelas_a = [85, 90, 78, 92]
total_a       = sum(nilai_kelas_a)
rata_a        = total_a / len(nilai_kelas_a)
print(f"Kelas A — Rata-rata: {rata_a:.1f}, Tertinggi: {max(nilai_kelas_a)}")

nilai_kelas_b = [70, 85, 88, 76]
total_b       = sum(nilai_kelas_b)
rata_b        = total_b / len(nilai_kelas_b)
print(f"Kelas B — Rata-rata: {rata_b:.1f}, Tertinggi: {max(nilai_kelas_b)}")

# ── SESUDAH: fungsi yang bisa dipakai ulang ───────────────
def laporan_nilai(nama_kelas, nilai):
    """Cetak laporan ringkas untuk satu kelas."""
    rata   = sum(nilai) / len(nilai)
    tinggi = max(nilai)
    print(f"Kelas {nama_kelas} — Rata-rata: {rata:.1f}, Tertinggi: {tinggi}")

laporan_nilai("A", [85, 90, 78, 92])
laporan_nilai("B", [70, 85, 88, 76])
laporan_nilai("C", [95, 88, 91, 87])  # tambah kelas baru = satu baris saja

6.2 Parameter, Argumen, dan Nilai Default

Fungsi yang lebih berguna bisa menerima input dari luar — data yang ingin kamu olah atau konfigurasi yang ingin kamu terapkan. Di sinilah parameter masuk. Parameter adalah nama-nama yang kamu deklarasikan di dalam tanda kurung saat mendefinisikan fungsi, sementara argumen adalah nilai nyata yang kamu kirimkan saat memanggil fungsi tersebut.

Perbedaan ini murni terminologi — dalam percakapan sehari-hari keduanya sering dipakai bergantian, dan itu tidak masalah. Yang penting kamu paham konsepnya: parameter adalah tempat penampungnya, argumen adalah isinya.

python 
# nama dan usia adalah PARAMETER
def perkenalan(nama, usia):
    print(f"Nama: {nama}, Usia: {usia} tahun")

# "Rina" dan 25 adalah ARGUMEN
perkenalan("Rina", 25)   # → Nama: Rina, Usia: 25 tahun
perkenalan("Dodi", 30)   # → Nama: Dodi, Usia: 30 tahun

# Argumen posisional vs argumen kata kunci
perkenalan("Sari", 22)              # posisional — urutan penting
perkenalan(nama="Sari", usia=22)    # kata kunci — urutan boleh dibalik
perkenalan(usia=22, nama="Sari")    # hasil sama persis

Nilai Default (Default Parameter Value)

Kamu bisa memberi nilai default untuk parameter — nilai yang dipakai jika argumen untuk parameter itu tidak diberikan saat pemanggilan. Ini sangat berguna untuk membuat fungsi yang fleksibel: bisa dipakai dengan cara sederhana tanpa banyak argumen, tapi juga bisa dikustomisasi sepenuhnya jika perlu. Dalam kecerdasan buatan, nilai default sangat sering muncul untuk hiperparameter model:

python 
# Nilai default: learning_rate dan epochs punya nilai bawaan
def inisialisasi_model(nama_model, learning_rate=0.001, epochs=100, verbose=True):
    """Inisialisasi konfigurasi pelatihan model."""
    if verbose:
        print(f"Model     : {nama_model}")
        print(f"LR        : {learning_rate}")
        print(f"Epochs    : {epochs}")
    return {"model": nama_model, "lr": learning_rate, "epochs": epochs}

# Pakai semua default
inisialisasi_model("ResNet50")

# Override sebagian
inisialisasi_model("ResNet50", learning_rate=0.01)

# Override semua
inisialisasi_model("EfficientNet", 0.0001, 50, verbose=False)
⚠️
Jangan Pakai List atau Dict Sebagai Nilai Default

Ini adalah jebakan klasik Python yang menggigit banyak pemula. Menulis def fungsi(data=[]) tampak tidak berbahaya, tapi list default itu dibagi di semua pemanggilan — bukan dibuat ulang setiap kali dipanggil. Kalau kamu memodifikasi list itu di dalam fungsi, modifikasi itu akan bertahan ke pemanggilan berikutnya. Solusinya: gunakan None sebagai default, lalu buat list baru di dalam fungsi jika nilainya None.

python — jebakan mutable default 
# ── SALAH: list sebagai default — bug tersembunyi ────────
def tambah_item_salah(item, keranjang=[]):
    keranjang.append(item)
    return keranjang

print(tambah_item_salah("apel"))     # → ['apel']
print(tambah_item_salah("mangga"))   # → ['apel', 'mangga']  ← harusnya ['mangga']!

# ── BENAR: gunakan None sebagai default ──────────────────
def tambah_item_benar(item, keranjang=None):
    if keranjang is None:
        keranjang = []   # list baru setiap kali dipanggil tanpa argumen
    keranjang.append(item)
    return keranjang

print(tambah_item_benar("apel"))     # → ['apel']
print(tambah_item_benar("mangga"))   # → ['mangga']  ← benar!

*args dan **kwargs — Parameter Fleksibel

Terkadang kamu tidak tahu sebelumnya berapa banyak argumen yang akan diterima fungsimu. Python punya dua mekanisme untuk ini. *args mengumpulkan semua argumen posisional ekstra menjadi sebuah tuple, sementara **kwargs mengumpulkan semua argumen kata kunci ekstra menjadi sebuah dictionary. Keduanya akan sering kamu lihat dalam kode library ML:

python 
# *args — jumlah argumen posisional tidak terbatas
def jumlahkan(*angka):
    """Jumlahkan semua angka yang diberikan."""
    print(f"Menerima {len(angka)} angka: {angka}")
    return sum(angka)

print(jumlahkan(1, 2))              # → 3
print(jumlahkan(1, 2, 3, 4, 5))    # → 15

# **kwargs — jumlah argumen kata kunci tidak terbatas
def cetak_config(**pengaturan):
    """Cetak semua pengaturan yang diberikan."""
    print("=== Konfigurasi Model ===")
    for kunci, nilai in pengaturan.items():
        print(f"  {kunci}: {nilai}")

cetak_config(model="XGBoost", lr=0.05, depth=6, subsample=0.8)

6.3 Nilai Kembalian (Return Value)

Fungsi bisa melakukan dua hal: menghasilkan efek samping (seperti mencetak ke layar atau menyimpan file) atau menghasilkan sebuah nilai kembalian yang bisa dipakai oleh kode lain. Kata kunci return dipakai untuk mengembalikan nilai dari fungsi ke tempat fungsi itu dipanggil. Setelah bertemu return, fungsi langsung berhenti — baris kode setelahnya tidak dijalankan.

Fungsi tanpa pernyataan return, atau yang hanya menulis return tanpa nilai, secara implisit mengembalikan None. Ini penting dipahami karena bug yang sering muncul adalah memakai hasil fungsi yang sebenarnya mengembalikan None.

python 
# Fungsi dengan return value
def celsius_ke_fahrenheit(c):
    """Konversi suhu dari Celsius ke Fahrenheit."""
    return (c * 9 / 5) + 32

suhu_f = celsius_ke_fahrenheit(100)
print(f"100°C = {suhu_f}°F")   # → 100°C = 212.0°F

# Nilai kembalian bisa langsung dipakai dalam ekspresi
print(celsius_ke_fahrenheit(37) > 98)   # → True

# Return awal — hentikan fungsi jika kondisi tidak terpenuhi
def bagi_aman(a, b):
    """Bagi a dengan b, kembalikan None jika b adalah nol."""
    if b == 0:
        print("Peringatan: pembagi tidak boleh nol")
        return None   # keluar lebih awal
    return a / b

print(bagi_aman(10, 2))   # → 5.0
print(bagi_aman(10, 0))   # → Peringatan: ... lalu None

Mengembalikan Banyak Nilai Sekaligus

Python memungkinkan sebuah fungsi mengembalikan lebih dari satu nilai sekaligus — sebenarnya yang terjadi adalah fungsi mengembalikan sebuah tuple, yang kemudian bisa langsung dibongkar ke beberapa variabel. Ini pola yang sangat umum dalam kode ML, terutama untuk fungsi evaluasi model yang biasanya mengembalikan beberapa metrik sekaligus:

python 
def statistik_dasar(data):
    """Hitung statistik dasar dari list angka."""
    n      = len(data)
    rata   = sum(data) / n
    minimum = min(data)
    maksimum = max(data)
    # varian sederhana
    varian = sum((x - rata) ** 2 for x in data) / n
    std    = varian ** 0.5
    return rata, minimum, maksimum, std   # mengembalikan tuple

nilai = [85, 92, 78, 95, 88, 70, 91]

# Terima sebagai tuple
hasil = statistik_dasar(nilai)
print(hasil)   # → (85.57..., 70, 95, 7.68...)

# Atau langsung bongkar ke variabel — cara yang lebih rapi
rata, minimum, maksimum, std = statistik_dasar(nilai)
print(f"Rata-rata: {rata:.1f}")
print(f"Rentang  : {minimum} – {maksimum}")
print(f"Std Dev  : {std:.2f}")

# Pola serupa untuk fungsi evaluasi model
def evaluasi_klasifikasi(y_asli, y_prediksi):
    """Hitung metrik evaluasi klasifikasi biner."""
    n         = len(y_asli)
    benar     = sum(1 for a, p in zip(y_asli, y_prediksi) if a == p)
    akurasi   = benar / n
    tp        = sum(1 for a, p in zip(y_asli, y_prediksi) if a == 1 and p == 1)
    fp        = sum(1 for a, p in zip(y_asli, y_prediksi) if a == 0 and p == 1)
    presisi   = tp / (tp + fp) if (tp + fp) > 0 else 0
    return akurasi, presisi

y_asli      = [1, 0, 1, 1, 0, 1]
y_prediksi  = [1, 0, 0, 1, 0, 1]
akurasi, presisi = evaluasi_klasifikasi(y_asli, y_prediksi)
print(f"Akurasi: {akurasi:.1%}, Presisi: {presisi:.1%}")
# → Akurasi: 83.3%, Presisi: 100.0%
✅
Fungsi yang Baik Selalu Mengembalikan Nilai yang Konsisten

Hindari fungsi yang di satu jalur mengembalikan nilai angka, dan di jalur lain tidak mengembalikan apapun (implisit None). Kode yang memanggil fungsimu tidak akan tahu harus berharap apa. Kalau fungsi mengembalikan angka, ia selalu harus mengembalikan angka — atau secara eksplisit mengembalikan None dan mendokumentasikannya di docstring.

6.4 Ruang Lingkup Variabel (Variable Scope)

Tidak semua variabel bisa diakses dari semua tempat dalam kode. Ruang lingkup *(scope)* menentukan di mana sebuah variabel "hidup" dan dari mana ia bisa diakses. Python mengikuti aturan LEGB — Python pertama mencari variabel di lingkup Lokal, lalu Enclosing, lalu Global, dan terakhir Bawaan *(built-in)*.

Yang paling penting untuk kamu pahami sekarang adalah perbedaan antara variabel lokal dan variabel global. Variabel yang dibuat di dalam fungsi adalah variabel lokal — ia hanya ada selama fungsi itu berjalan dan tidak bisa diakses dari luar. Variabel yang dibuat di luar fungsi adalah variabel global — ia bisa dibaca dari dalam fungsi, tapi tidak bisa diubah dari dalam fungsi tanpa kata kunci khusus.

python 
# Variabel lokal — hanya ada di dalam fungsi
def hitung():
    hasil = 42   # variabel lokal
    print(hasil)  # → 42

hitung()
# print(hasil)  # → NameError: 'hasil' tidak dikenal di sini

# Variabel global bisa DIBACA dari dalam fungsi
NAMA_MODEL = "ResNet50"   # variabel global (konvensi: HURUF_BESAR)

def info_model():
    print(f"Model yang dipakai: {NAMA_MODEL}")   # membaca global — OK

info_model()   # → Model yang dipakai: ResNet50

# Variabel global TIDAK BISA diubah langsung dari dalam fungsi
jumlah = 0

def tambah_satu_salah():
    jumlah = jumlah + 1   # UnboundLocalError! Python anggap ini variabel lokal baru

# Solusinya ada dua:
# 1. Gunakan kata kunci 'global' (tidak dianjurkan untuk kode serius)
def tambah_satu_global():
    global jumlah
    jumlah += 1

# 2. Cara yang lebih baik: kembalikan nilai baru, jangan ubah global
def tambah_satu_bersih(n):
    return n + 1   # terima input, kembalikan output — tidak sentuh global

jumlah = tambah_satu_bersih(jumlah)
print(jumlah)   # → 1
⚠️
Hindari Variabel Global yang Diubah-Ubah

Fungsi yang membaca dan mengubah variabel global membuat kode sangat sulit dilacak — kamu tidak bisa tahu kondisi program hanya dari membaca fungsinya saja. Fungsi yang baik adalah fungsi murni *(pure function)*: semua input datang dari parameter, semua output melalui return value. Tidak ada ketergantungan tersembunyi pada kondisi luar. Kode ML yang terstruktur dengan baik hampir selalu mengikuti prinsip ini.

Scope dalam Fungsi Bersarang

Python juga mendukung fungsi yang didefinisikan di dalam fungsi lain — disebut fungsi bersarang *(nested function)*. Fungsi dalam bisa mengakses variabel dari fungsi luar melalui lingkup yang disebut enclosing scope. Ini adalah dasar dari konsep closure yang akan kamu lihat lebih dalam di bab-bab mendatang. Untuk sekarang, cukup ketahui bahwa ini ada:

python 
# Fungsi bersarang — fungsi dalam bisa baca variabel fungsi luar
def buat_penambah(n):
    """Buat fungsi yang menambahkan n ke input apapun."""
    def penambah(x):
        return x + n    # 'n' dari fungsi luar — enclosing scope
    return penambah    # kembalikan fungsi itu sendiri!

tambah_10 = buat_penambah(10)
tambah_5  = buat_penambah(5)

print(tambah_10(3))   # → 13
print(tambah_5(3))    # → 8
print(tambah_10(7))   # → 17

# tambah_10 dan tambah_5 adalah 'closure' — fungsi yang membawa
# serta nilai dari enclosing scope ke manapun ia pergi

6.5 Fungsi sebagai Objek Pertama — Pengantar

Ini adalah konsep yang mungkin terasa sedikit aneh di awal, tapi sangat penting untuk pemrograman Python tingkat lanjut. Di Python, fungsi adalah objek — sama seperti integer, string, atau list. Artinya: fungsi bisa disimpan dalam variabel, dimasukkan ke dalam list, dioper sebagai argumen ke fungsi lain, atau dikembalikan dari fungsi lain. Bahasa yang mendukung ini disebut mendukung fungsi sebagai "objek kelas pertama" *(first-class citizen)*.

Kamu sebenarnya sudah melihat ini di bab sebelumnya — fungsi bawaan Python seperti sorted() bisa menerima argumen key yang nilainya adalah fungsi lain. Di dunia kecerdasan buatan, konsep ini muncul setiap hari: callback pelatihan, fungsi aktivasi yang dioper ke lapisan neural network, pipeline transformasi data, dan sebagainya.

python 
# Fungsi bisa disimpan dalam variabel
def salam(nama):
    return f"Halo, {nama}!"

ucapan = salam    # tanpa tanda kurung — ini menyimpan fungsinya, bukan memanggilnya
print(ucapan("Rina"))   # → Halo, Rina!
print(type(ucapan))      # → <class 'function'>

# Fungsi bisa dimasukkan ke dalam list
def kuadrat(x):   return x ** 2
def kubik(x):     return x ** 3
def akar(x):      return x ** 0.5

transformasi = [kuadrat, kubik, akar]   # list berisi fungsi!
for f in transformasi:
    print(f"{f.__name__}(4) = {f(4)}")
# → kuadrat(4) = 16
# → kubik(4) = 64
# → akar(4) = 2.0

Mengoper Fungsi sebagai Argumen

Ini salah satu penggunaan yang paling sering kamu lihat. Fungsi yang menerima fungsi lain sebagai argumen disebut fungsi tingkat tinggi *(higher-order function)*. Contoh klasiknya adalah fungsi bawaan sorted(), map(), dan filter():

python 
# sorted() menerima fungsi sebagai argumen 'key'
model_results = [
    {"nama": "XGBoost",         "akurasi": 0.93},
    {"nama": "Random Forest",     "akurasi": 0.91},
    {"nama": "Regresi Logistik",   "akurasi": 0.82},
]

# Fungsi untuk mengambil nilai yang dipakai sebagai kunci urutan
def ambil_akurasi(item):
    return item["akurasi"]

urut_terbaik = sorted(model_results, key=ambil_akurasi, reverse=True)
for m in urut_terbaik:
    print(f"{m['akurasi']:.1%} — {m['nama']}")
# → 93.0% — XGBoost
# → 91.0% — Random Forest
# → 82.0% — Regresi Logistik

# Fungsi kustom yang menerima fungsi lain sebagai argumen
def terapkan_ke_semua(data, fungsi_transformasi):
    """Terapkan fungsi ke setiap elemen dalam data."""
    return [fungsi_transformasi(x) for x in data]

nilai = [4, 9, 16, 25]
print(terapkan_ke_semua(nilai, akar))     # → [2.0, 3.0, 4.0, 5.0]
print(terapkan_ke_semua(nilai, kuadrat))  # → [16, 81, 256, 625]

Lambda — Fungsi Anonim Satu Baris

Lambda adalah cara membuat fungsi kecil sekali pakai tanpa nama, langsung di tempat ia dibutuhkan. Sintaksinya adalah lambda parameter: ekspresi. Lambda berguna ketika fungsi yang dibutuhkan sangat sederhana — hanya satu ekspresi — dan tidak perlu punya nama karena hanya akan dipakai sekali:

python 
# Fungsi biasa vs lambda — hasil identik
def kali_dua(x):
    return x * 2

kali_dua_lambda = lambda x: x * 2

print(kali_dua(5))         # → 10
print(kali_dua_lambda(5))  # → 10

# Lambda paling berguna langsung sebagai argumen fungsi lain
data = ["apel", "mangga", "jeruk", "pisang"]

# Urutkan berdasarkan panjang nama buah
urut_panjang = sorted(data, key=lambda x: len(x))
print(urut_panjang)   # → ['apel', 'jeruk', 'mangga', 'pisang']

# Urutkan model berdasarkan akurasi — lebih ringkas dari sebelumnya
urut_terbaik_2 = sorted(model_results, key=lambda m: m["akurasi"], reverse=True)

# Lambda dengan beberapa parameter
hitung_skor = lambda presisi, ingatan: 2 * (presisi * ingatan) / (presisi + ingatan)
print(f"F1-score: {hitung_skor(0.85, 0.78):.3f}")
# → F1-score: 0.813
💡
Lambda Hanya untuk Ekspresi Sederhana

Lambda hanya bisa berisi satu ekspresi — tidak bisa berisi pernyataan seperti if-else yang berdiri sendiri, for loop, atau return. Kalau logikanya lebih kompleks dari satu ekspresi, buat fungsi biasa dengan def. Kode yang mudah dibaca lebih berharga daripada kode yang terlihat pintar tapi sulit dipahami.

Konsep Bentuk Kapan Dipakai
Fungsi biasa def nama(param): ... Logika apapun, perlu didokumentasikan, dipanggil berulang
Fungsi dengan default def f(x, n=10): ... Argumen opsional dengan nilai bawaan yang masuk akal
*args def f(*args): ... Jumlah argumen posisional tidak diketahui sebelumnya
**kwargs def f(**kwargs): ... Jumlah argumen kata kunci tidak diketahui sebelumnya
Lambda lambda x: ekspresi Fungsi sekali pakai sederhana, biasanya sebagai argumen
